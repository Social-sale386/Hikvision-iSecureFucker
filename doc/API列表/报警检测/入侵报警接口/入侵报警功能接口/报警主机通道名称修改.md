# 报警主机通道名称修改

> 接口说明
> 1、根据通道编号修改通道名称；
> 2、只会修改平台里通道名称，不涉及下发设备；
> 3、仅支持修改入侵报警管理的设备，暂不支持修改简单设备管理的第三方厂商设备。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/scpms/v1/channels/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/scpms/v1/channels/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`channelType`|`body`|`string`|是|通道类型，子系统：subSys，防区：defence|
|`channelIndexCode`|`body`|`string`|是|通道indexCode，可从查询入侵报警主机通道列表v2 接口获取返回值indexCode|
|`channelName`|`body`|`string`|是|通道名称(最大支持32位，不能包含’ / \ : * ? “” < >|

## 请求参数示例
```json
{
  "channelType": "subSys",
  "channelIndexCode": "5fa7cdca292249e3a541a6e794f5707f",
  "channelName": "通道1"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.4.1 入侵报警错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|返回结果，true-成功，false-失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": true
}
```
