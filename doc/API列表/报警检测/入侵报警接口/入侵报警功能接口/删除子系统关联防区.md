# 删除子系统关联防区

> 接口说明
> 1、关联了子系统的防区，可通过此接口解除防区和子系统的关联关系；
> 2、入参中如果包含不存在的子系统、防区编码，直接返回失败；
> 3、单个子系统最大支持256个防区；
> 4、仅支持修改入侵报警管理的设备，暂不支持修改简单设备管理的第三方厂商设备 。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/scpms/v1/subsystem/defences/delete`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/scpms/v1/subsystem/defences/delete
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`subSystemIndexCode`|`body`|`string`|是|子系统编码，可从查询入侵报警主机通道列表v2 接口获取返回值indexCode。|
|`defenceIndexCodes`|`body`|`string[]`|是|防区编码列表|

## 请求参数示例
```json
{
  "subSystemIndexCode": "5fa7cdca292249e3a541a6e794f5707f",
  "defenceIndexCodes": [
    "242b84ceae2f483cace50df7ed7acaab"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.4.1 入侵报警错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回结果，成功则data为null|
|`+total`|`body`|`number`|否|数据记录总数|
|`+list`|`body`|`object[]`|否|返回数据列表|
|`++indexCode`|`body`|`string`|否|删除失败的子系统编码|
|`++success`|`body`|`boolean`|否|操作结果，true – 成功， false - 失败|
|`++errorCode`|`body`|`string`|否|错误码，参考附录E.4.1 入侵报警错误码|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": null
}
```
