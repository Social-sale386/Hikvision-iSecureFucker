# 添加人员v2

> 接口说明
> 添加人员信息接口，注意，在安保基础数据配置的必选字段必须都包括在入参中。
> 人员添加的时候，可以指定人员personId，不允许与其他人员personId重复，包括已删除的人员。
> 本接口支持人员信息的扩展字段，按照属性定义key:value上传即可， 可通过获取资源属性接口，获取平台已启用的人员属性信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/resource/v2/person/single/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v2/person/single/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personId`|`body`|`string`|否|人员ID，可以指定人员personId，不允许与其他人员personId重复，包括已删除的人员。 为空时平台自动生成人员ID|
|`personName`|`body`|`string`|是|人员名称，1~32个字符；不能包含 ’ / \ : * ? " < >|
|`orgIndexCode`|`body`|`string`|是|所属组织标识，必须是已存在组织，从获取组织列表接口获取返回参数orgIndexCode|
|`birthday`|`body`|`string`|否|出生日期，举例：1992-09-12|
|`phoneNo`|`body`|`string`|否|手机号，1-20位数字,平台上人员信息实名标识选择为手机号码时必填|
|`email`|`body`|`string`|否|邮箱，举例：hic@163.com|
|`certificateType`|`body`|`string`|否|证件类型，参考附录A 数据字典，平台上人员信息实名标识选择为身份证件时必填|
|`certificateNo`|`body`|`string`|否|证件号码，1-20位数字字母，平台上人员信息实名标识选择为身份证件时必填|
|`jobNo`|`body`|`string`|否|工号，1-32个字符|
|`faces`|`body`|`object[]`|否|人脸信息|
|`+faceData`|`Body`|`string`|是|人脸图片base64编码后的字符。|

补充说明:
- 建议：
- 1. 正面免冠照，露出眼睛和眉毛；
- 2. 图片文件支持jpg 格式；
- 3. 图片文件大小10 KB~200 KB。

## 请求参数示例
```json
{
  "personName": "person0",
  "orgIndexCode": "1rwad89d-0ce6-4826-9146-6b71f037d81e",
  "birthday": "1990-01-01",
  "phoneNo": "13000110011",
  "email": "person1@person.com",
  "certificateType": "111",
  "certificateNo": "11001001001100110011",
  "jobNo": "111111",
  "faces": [
    {
      "faceData": "/9j/4AAQSkZJRgABAQEAAAAAAAD/4QBCRXhpZgAATU.."
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回说明|
|`data`|`body`|`object`|否|成功添加的人员信息|
|`+personId`|`body`|`string`|否|成功添加的人员Id|
|`+faceId`|`body`|`string`|否|成功添加的人脸Id|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "personId": "a8f70815c841u5av6136v191u0c",
    "faceId": "4233319c-74cc-4dfc-a0c2-a71609210518"
  }
}
```
