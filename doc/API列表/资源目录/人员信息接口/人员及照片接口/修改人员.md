# 修改人员

> 接口说明
> 根据人员编号修改人员信息。本接口支持人员信息的扩展字段，按照属性定义key:value上传即可， 可通过获取资源属性接口，获取平台已启用的人员属性信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/person/single/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/person/single/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personId`|`body`|`string`|是|人员id|
|`personName`|`body`|`string`|否|人员名称，1~32个字符；不能包含 ’ / \ : * ? " < >|
|`orgIndexCode`|`body`|`string`|否|所属组织标识，必须是已存在组织，从获取组织列表接口获取返回参数orgIndexCode|
|`birthday`|`body`|`string`|否|出生日期，举例：1992-09-12|
|`phoneNo`|`body`|`string`|否|手机号，1-20位数字|
|`email`|`body`|`string`|否|邮箱，举例：hic@163.com|
|`certificateType`|`body`|`string`|否|证件类型，参考附录A 数据字典|
|`certificateNo`|`body`|`string`|否|证件号码，1-20位数字字母|
|`jobNo`|`body`|`string`|否|工号，，1-32个字符|

## 请求参数示例
```json
{
  "personId": "f5110a5f05ec4a66afa0754dca482522",
  "personName": "person0",
  "orgIndexCode": "88602c10-6904-4867-b556-9cd3721f4281",
  "birthday": "1990-01-01",
  "phoneNo": "13000110011",
  "email": "person1@person.com",
  "certificateType": "111",
  "certificateNo": "11001001001100110011",
  "jobNo": "111"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`string`|否|成功则data为空|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": null
}
```
