# 查询人员列表v2

> 接口说明
> 查询人员列表接口可以根据人员ID集、人员姓名、人员性别、所属组织、证件类型、证件号码、人员状态这些查询条件来进行高级查询；若不指定查询条件，即全量获取所有的人员信息。返回结果分页展示。
> 注：若指定多个查询条件，表示将这些查询条件进行”与”的组合后进行精确查询。
> 根据”人员名称personName”查询为模糊查询。本接口支持自定义属性的返回， 通过获取资源属性接口获取平台已支持人员属性的说明。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v2/person/advance/personList`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v2/person/advance/personList
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personIds`|`body`|`string`|否|人员ID集合|
|`personName`|`body`|`string`|否|人员姓名|
|`orgIndexCodes`|`body`|`string`|否|所属组织唯一标识码集合|
|`certificateType`|`body`|`integer`|否|证件类型，参考附录A 数据字典|
|`certificateNo`|`body`|`string`|否|证件号码|
|`pageNo`|`body`|`integer`|是|当前页码需大于0（pageNo>0）|
|`pageSize`|`body`|`integer`|是|每页记录展示的数目应大于0，小于等于1000（0<pageSize<=1000）|
|`isSubOrg`|`body`|`boolean`|否|是否包含下级组织，true时，搜索orgIndexCodes及其所有子孙组织的人员；|
|`cardNo`|`body`|`string`|否|卡号|
|`plateNo`|`body`|`string`|否|车牌号|
|`orderBy`|`body`|`string`|否|排序字段，注意：排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序：desc|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|否|资源属性名，key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|否|操作运算符， 0 ：=， 1 ：>=， 2 ：<=， 3 ：in， 4 ：not in， 5 ：between， 6 ：like， 7 ：pre like， 8 ：suffix like|
|`+values`|`Body`|`string[]`|否|资源属性值，=、>=、<=、like、values数组长度只能是1； in、not in，values数组长度大于1，最大不超时20； in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到； between只能用于整形、日期 ； like只能用于字符串。|

补充说明:
- 多个值使用英文逗号分隔，不超过1000个
- 支持中英文字符，不能包含 ’ / \ : * ? " < >
- 多个值使用英文逗号分隔，不超过1000个
- false时，只搜索orgIndexCodes的人员
- 升序：asc

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|人员信息结构体|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|实际每页记录总数|
|`+list`|`body`|`object[]`|否|人员列表|
|`++personId`|`body`|`string`|否|人员ID|
|`++personName`|`body`|`string`|否|姓名|
|`++orgPath`|`body`|`string`|否|所属组织目录，@进行分割，例如 @root000000@|
|`++orgIndexCode`|`body`|`string`|否|所属组织唯一标识码|
|`++orgPathName`|`body`|`string`|否|所属组织名称，@进行分割，如@默认部门@|
|`++certificateType`|`body`|`number`|否|证件类型，参考附录A 数据字典|
|`++certificateNo`|`body`|`string`|否|证件号码|
|`++createTime`|`body`|`string`|否|创建时间，|
|`++updateTime`|`body`|`string`|否|更新时间，|
|`++phoneNo`|`body`|`string`|否|联系电话|
|`++jobNo`|`body`|`string`|否|工号|
|`++personPhoto`|`body`|`object`|否|人员图片信息|
|`+++personPhotoIndexCode`|`body`|`string`|否|人脸图片ID|
|`+++personId`|`body`|`string`|否|人员ID|
|`+++picUri`|`body`|`string`|否|图片相对url,通过提取人员图片接口获取图片数据|
|`+++serverIndexCode`|`body`|`string`|否|图片服务器唯一标示,提取人员图片接口必要入参|

补充说明:
- 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00
- 表示北京时间2017年7月26日21时30分08秒
- 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00
- 表示北京时间2017年7月26日21时30分08秒
