# 获取组织下人员列表v2

> 接口说明
> 根据组织编号获取组织下的人员信息列表，返回结果分页展示。本接口支持自定义属性的返回， 通过获取资源属性接口获取平台已支持人员属性的说明。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v2/person/orgIndexCode/personList`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v2/person/orgIndexCode/personList
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orgIndexCode`|`body`|`string`|是|组织唯一标识码，从获取组织列表接口获取返回参数orgIndexCode|
|`pageNo`|`body`|`integer`|是|当前页码（32）|
|`pageSize`|`body`|`integer`|是|实际每页记录总数（32）|

补充说明:
- 当前页码需大于0（pageNo>0）
- 每页记录展示的数目应大于0，小于等于1000（0<pageSize<=1000）

## 请求参数示例
```json
{
  "orgIndexCode": "ac7114be-c23a-49a2-96bf-52e2f12311d5",
  "pageNo": 1,
  "pageSize": 1000
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|人员信息结构体|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|每页记录总数|
|`+list`|`body`|`object[]`|否|人员列表|
|`++personId`|`body`|`string`|否|人员ID|
|`++personName`|`body`|`string`|否|姓名|
|`++orgIndexCode`|`body`|`string`|否|所属组织唯一标识码|
|`++phoneNo`|`body`|`string`|否|联系电话|
|`++jobNo`|`body`|`string`|否|工号|
|`++certificateType`|`body`|`number`|否|证件类型，参考附录A 数据字典|
|`++certificateNo`|`body`|`string`|否|证件号码|
|`++createTime`|`body`|`string`|否|创建时间，|
|`++updateTime`|`body`|`string`|否|更新时间，|
|`++orgPath`|`body`|`string`|否|所属组织目录，@进行分割，例如|
|`++orgPathName`|`body`|`string`|否|所属组织名称，@进行分割，如@默认部门@|
|`++personPhoto`|`body`|`object`|否|人员图片信息|
|`+++personPhotoIndexCode`|`body`|`string`|否|人脸图片ID|
|`+++picUri`|`body`|`string`|否|图片相对url,通过提取人员图片接口获取图片数据|
|`+++serverIndexCode`|`body`|`string`|否|图片服务器唯一标示,提取人员图片接口必要入参|
|`+++personId`|`body`|`string`|否|人员ID|

补充说明:
- 一串数字字符串
- 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00
- 表示北京时间2017年7月26日21时30分08秒
- 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00
- 表示北京时间2017年7月26日21时30分08秒
- @root000000@
