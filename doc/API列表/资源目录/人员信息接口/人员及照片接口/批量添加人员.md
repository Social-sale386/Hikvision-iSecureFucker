# 批量添加人员

> 接口说明
> 添加人员信息接口，注意，在安保基础数据配置的必选字段必须都包括在入参中。
> 批量添加仅支持人员基础属性。
> 人员批量添加的时候，可以指定人员personId，不允许与其他人员personId重复，包括已删除的人员。 如果用户不想指定personId，那么需要指定clientId，请求内的每一条数据的clientid必须唯一， 返回值会将平台生成的personid与clientid做绑定。
> 若personId和clientId都不指定，则无法准确判断哪部分人员添加成功。
> 本接口支持人员信息的扩展字段，按照属性定义key:value上传即可， 可通过获取资源属性接口，获取平台已启用的人员属性信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/person/batch/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/person/batch/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`Array[0]`|`body`|`object`|是|待添加的人员列表，单次操作上限为1000条|
|`+clientId`|`Body`|`number`|否|人员临时标志，当调用方未指定personId时，clientId作为人员标志，在返回值中标志每个人员的添加结果。 所以每次接口调用，每个人员的clientid需保持唯一，1~10个字符，只支持数字。 注：clientid只对本次调用有效|
|`+personId`|`Body`|`string`|否|唯一标志，可以指定人员personId，不允许与其他人员personId重复，包括已删除的人员。 为空时平台自动生成唯一标志|
|`+personName`|`Body`|`string`|是|人员名称，1~32个字符；不能包含 ’ / \ : * ? " < >|
|`+orgIndexCode`|`Body`|`string`|是|所属组织标识，必须是已存在组织|
|`+birthday`|`Body`|`string`|否|出生日期，举例：1992-09-12|
|`+phoneNo`|`Body`|`string`|否|手机号，1-20位数字,平台上人员信息实名标识选择为手机号码时必填|
|`+email`|`Body`|`string`|否|邮箱，举例：hic@163.com|
|`+certificateType`|`Body`|`string`|否|证件类型，参考附录A 数据字典，平台上人员信息实名标识选择为身份证件时必填|
|`+certificateNo`|`Body`|`string`|否|证件号码，1-20位数字字母，平台上人员信息实名标识选择为身份证件时必填|
|`+jobNo`|`Body`|`string`|否|工号，信息实名标识选择为工号时必填|

## 请求参数示例
```json
[
  {
    "clientId": 1,
    "personId": "y6a9fy8yh8cy0y010yc92y9y19c09",
    "personName": "person0",
    "orgIndexCode": "88602c10-6904-4867-b556-9cd3721f4281",
    "birthday": "1990-01-01",
    "phoneNo": "13000110011",
    "email": "person1@person.com",
    "certificateType": "111",
    "certificateNo": "11001001001100110011",
    "jobNo": "111"
  }
]
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回结果|
|`+successes`|`body`|`object[]`|否|批量添加成功的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++personId`|`body`|`string`|否|服务端生成的唯一标识|
|`+failures`|`body`|`object[]`|否|批量添加失败的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++code`|`body`|`string`|否|错误码|
|`++message`|`body`|`string`|否|错误信息|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "successes": [
      {
        "clientId": 123456789,
        "personId": "5668f44c-ae23-4674-9b79-d8f79f77fd41"
      }
    ],
    "failures": [
      {
        "clientId": 555,
        "code": "-1",
        "message": "OrgIndexCode Not Exists"
      }
    ]
  }
}
```
