# 生成卡片二维码

> 接口说明
> 用于生产卡片二维码，二维码默认有效期为24*60分钟，默认最大开锁次数4次.

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/cis/v1/card/barCode`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/cis/v1/card/barCode
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personId`|`body`|`string`|否|人员ID，cardNo存在的情况下 人员ID可不填，根据获取人员列表v2接口获取返回参数personId|
|`cardNo`|`body`|`string`|否|卡号|
|`certType`|`body`|`integer`|否|证件类型，当人员唯一标示是证件类型+证件号码时必填，111:身份证,414:护照,113:户口簿,335:驾驶证,131:工作证,133:学生证,990:其他|
|`certNum`|`body`|`string`|否|证件号码，当人员唯一标示是证件类型+证件号码时必填，根据获取人员列表v2接口获取返回参数certificateNo|
|`phone`|`body`|`string`|否|手机号码，当人员唯一标示是手机号码时必填，根据获取人员列表v2接口获取返回参数phoneNo|
|`employeeId`|`body`|`string`|否|工号，当人员唯一标示是工号时必填，根据获取人员列表v2接口获取返回参数jobNo|
|`duration`|`body`|`integer`|否|人员卡片二维码有效期时长，非必填字段，单位为分钟，范围为[5~72460]分钟，不传默认为24*60分钟|
|`maxLock`|`body`|`integer`|否|最大开锁次数，其中针对同一台设备最大开锁次数不超过指定的次数，不传默认最大开锁次数为4次|

## 请求参数示例
```json
{
  "personId": "f6c70bfeebc14838854f2f14620ab5ee",
  "cardNo": "1000000001",
  "certType": 111,
  "certNum": "330231232132132314481",
  "phone": "18969145145",
  "employeeId": "12345678",
  "duration": 1440,
  "maxLock": 4
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码,0 – 成功，其他- 失败，参考附录E.other.3 卡片错误码|
|`msg`|`body`|`string`|否|返回描述（记录接口执行情况说明信息）|
|`data`|`body`|`object`|否|返回数据|
|`+cardNo`|`body`|`string`|否|卡号|
|`+barCode`|`body`|`string`|否|二维码，将访客二维码内容转成二维码图片即可使用。二维码图片的生成采用QRCode编码、纠错等级为L，版本大于等于5|

```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "cardNo": "555001",
    "barCode": "VjAwMSvmZ0B8wU8Ij+mM/nHCLKac6AFi0zeBQO7cT2V4H6fLhRAjUNCauXF6dt4iMHCYZjo="
  }
}
```
