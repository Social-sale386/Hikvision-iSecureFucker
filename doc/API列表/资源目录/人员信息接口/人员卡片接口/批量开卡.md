# 批量开卡

> 接口说明
> 该接口主要是应用于对多个人同时开卡的场景，输入卡片开始有效日期、卡片截止有效日期以及对应的人员、卡片关联列表，实现对多个人员同时开卡的功能，开卡成功后，可以到相应子系统开启卡片的权限，例如到门禁子系统开启人员门禁权限。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/cis/v1/card/bindings`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/cis/v1/card/bindings
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`startDate`|`body`|`string`|否|卡片开始有效时间，时间格式：yyyy-MM-dd|
|`endDate`|`body`|`string`|否|卡片截止有效时间,时间格式：yyyy-MM-dd|
|`cardList`|`body`|`object[]`|是|待开卡列表,最大支持50张卡，为空返回公共错误码，超过范围返回业务错误码，参考附录E.other.3 卡片错误码|
|`+cardNo`|`Body`|`string`|是|卡号，最大长度64，建议是由读卡器直接读出来的，如果是三方系统同步过来，需要提前调研卡号的规则所购买的设备上是否兼容|
|`+personId`|`Body`|`string`|是|人员唯一标示， 根据获取人员列表v2接口获取返回参数personId|
|`+orgIndexCode`|`Body`|`string`|否|人员所属组织IndexCode，不传默认使用根组织|
|`+cardType`|`Body`|`number`|否|卡片类型,1：IC卡；2：CPU卡；3：远距离卡；4：M1卡，默认1-IC卡|

补充说明:
- 不能大于2037年12月31日

## 请求参数示例
```json
{
  "startDate": "2018-10-30",
  "endDate": "2018-10-30",
  "cardList": [
    {
      "cardNo": "100000001",
      "personId": "370d303b-3294-428b-993b-07e6b3f09295",
      "orgIndexCode": "root000000",
      "cardType": 1
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码,0 – 成功，其他- 失败，参考附录E.other.3 卡片错误码|
|`msg`|`body`|`string`|否|返回描述（记录接口执行情况说明信息）|
|`data`|`body`|`object[]`|否|返回数据|
|`+cardId`|`body`|`string`|否|卡片ID|
|`+personId`|`body`|`string`|否|人员ID|
|`+orgIndexCode`|`body`|`string`|否|人员组织IndexCode|
|`+cardNo`|`body`|`string`|否|卡号|
