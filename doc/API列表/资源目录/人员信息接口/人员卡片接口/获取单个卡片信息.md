# 获取单个卡片信息

> 接口说明
> 获取卡片列表接口可用来全量同步卡片信息，返回结果分页展示，不作权限过滤。
> 注：卡号为精确查找

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/irds/v1/card/cardInfo`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/irds/v1/card/cardInfo
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`cardNo`|`body`|`string`|否|卡片号码，卡片号码和卡片ID两者同时只能传一种参数，根据获取卡片列表接口获取返回参数cardNo|
|`cardId`|`body`|`string`|否|卡号ID，卡片号码和卡片ID两者同时只能传一种参数，根据获取卡片列表接口获取返回参数cardId|

## 请求参数示例
```json
{
  "cardNo": "12345678"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码,0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|卡片信息结构体|
|`+cardId`|`body`|`string`|否|卡片ID|
|`+cardNo`|`body`|`string`|否|卡号|
|`+personId`|`body`|`string`|否|持卡人员id|
|`+personName`|`body`|`string`|否|持卡人名称|
|`+useStatus`|`body`|`number`|否|使用状态标记|
|`+startDate`|`body`|`string`|否|生效日期，|
|`+endDate`|`body`|`string`|否|失效日期，|
|`+lossDate`|`body`|`string`|否|挂失时间，|
|`+unlossDate`|`body`|`string`|否|解除挂失时间，|
