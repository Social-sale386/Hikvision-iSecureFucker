# 查询卡片列表

> 接口说明
> 查询卡片列表接口可以根据卡片号码、人员姓名、卡片状态、人员ID集合等查询条件来进行高级查询；若不指定查询条件，即全量获取所有的卡片信息。返回结果分页展示。
> 注：若指定多个查询条件，表示将这些查询条件进行“与”的组合后进行查询。
> “人员名称personName”条件查询为模糊查询。
> “卡片号码cardNo”条件查询为模糊查询。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/irds/v1/card/advance/cardList`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/irds/v1/card/advance/cardList
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`pageNo`|`body`|`integer`|是|当前页码，pageNo≥1|
|`pageSize`|`body`|`integer`|是|分页大小，0<pageSize≤1000|
|`personName`|`body`|`string`|否|人员姓名,模糊查询|
|`cardNo`|`body`|`string`|否|卡号,模糊查询， 根据获取卡片列表接口获取返回参数cardNo|
|`useStatus`|`body`|`integer`|否|使用状态标记，1：正常|
|`personIds`|`body`|`string`|否|人员ID集，多个值使用英文逗号分隔，不超过1000个， 根据获取人员列表v2接口获取返回参数personId|
|`orderBy`|`body`|`string`|否|排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序：desc|

补充说明:
- 2：挂失
- 升序：asc

## 请求参数示例
```json
{
  "pageNo": 1,
  "pageSize": 1000,
  "personName": "张三",
  "cardNo": "1321424",
  "useStatus": 1,
  "personIds": "2334565",
  "orderBy": "cardNo",
  "orderType": "asc"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码,0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|

|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|分页大小|
|`+list`|`body`|`object[]`|否|返回卡片数据列表|
|`++cardId`|`body`|`string`|否|卡片ID|
|`++cardNo`|`body`|`string`|否|卡号|
|`++personId`|`body`|`string`|否|持卡人员id|
|`++personName`|`body`|`string`|否|持卡人名称|
|`++useStatus`|`body`|`number`|否|使用状态标记，1：正常|
|`++startDate`|`body`|`string`|否|生效日期，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++endDate`|`body`|`string`|否|失效日期，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++lossDate`|`body`|`string`|否|挂失时间，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++unlossDate`|`body`|`string`|否|解除挂失时间，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++description`|`body`|`string`|否|描述|
|`++createTime`|`body`|`string`|否|创建时间，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++updateTime`|`body`|`string`|否|修改时间，要求遵守ISO8601标准，如2018-07-26T21:30:08+08:00|
|`++cardType`|`body`|`number`|否|卡类型|
|`++orgIndexCode`|`body`|`string`|否|所属组织|
|`++isBindFinger`|`body`|`number`|否|是否绑定指纹|
|`++isBindFace`|`body`|`number`|否|是否绑定人脸|
