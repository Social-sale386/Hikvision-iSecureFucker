# 卡片退卡

> 接口说明
> 该接口主要是应用于对人员下卡片进行退卡，输入卡号以及所属人员id，实现卡片退卡的功能。退卡成功后，相应子系统的卡片权限清除，例如所属卡片在门禁子系统的门禁权限全部清除。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/cis/v1/card/deletion`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/cis/v1/card/deletion
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`cardNumber`|`body`|`string`|是|卡号，根据获取卡片列表接口获取返回参数cardNo|
|`personId`|`body`|`string`|是|人员唯一标示，根据获取人员列表v2接口获取返回参数personId|

## 请求参数示例
```json
{
  "cardNumber": "100000002",
  "personId": "370d303b-3294-428b-993b-07e6b3f09295"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码,0 – 成功，其他- 失败，参考附录E.other.3 卡片错误码|
|`msg`|`body`|`string`|否|返回描述（记录接口执行情况说明信息）|
|`data`|`body`|`string`|否|null|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": null
}
```
