# 批量添加组织

> 接口说明
> 添加组织信息。
> 支持三方指定组织唯一标识， 也支持ISC独立生成组织唯一标识。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/org/batch/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/org/batch/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`Array[0]`|`body`|`object`|是|待添加的组织列表，单次提交上限为1000条，其他已定义的组织属性，按照key:value上传即可。|
|`+clientId`|`Body`|`number`|否|调用方指定标识，接口执行成功后将服务端生成的标识与此标识绑定后返回，再通过返回值中的clientId判断哪些成功，哪些失败。所以建议每次接口调用，clientid保持唯一。|
|`+orgIndexCode`|`Body`|`string`|否|组织唯一标志，不允许与其它组织唯一标志重复，包括已删除的组织，值为空或者不传此字段系统自动生成唯一标志|
|`+orgName`|`Body`|`string`|是|组织名称，1~32个字符；不能包含 ’ / \ : * ? " < >|
|`+parentIndexCode`|`Body`|`string`|是|父组织唯一标识码|
|`+orgCode`|`Body`|`string`|否|组织编码，当添加小区节点时必填，编码使用01101开头的8位数字编码，当添加楼栋单元时必填，编码使用01101开头的20位数字编码，|

补充说明:
- 注：clientid只对本次调用有效
- 其它场景下改值无效

## 请求参数示例
```json
[
  {
    "clientId": 123456789,
    "orgIndexCode": "1239017947124",
    "orgName": "组织test1",
    "parentIndexCode": "root000000"
  }
]
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回结果|
|`+successes`|`body`|`object[]`|否|批量添加成功的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++orgIndexCode`|`body`|`string`|否|服务端生成的唯一标识|
|`+failures`|`body`|`object[]`|否|批量添加失败的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++code`|`body`|`string`|否|错误码|
|`++msg`|`body`|`string`|否|错误信息|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "successes": [
      {
        "clientId": 1,
        "orgIndexCode": "qyc8c894ch1y94c19y4c2"
      }
    ],
    "failures": [
      {
        "clientId": 22,
        "code": "-1",
        "msg": "error"
      }
    ]
  }
}
```
