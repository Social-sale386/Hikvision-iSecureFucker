# 查询组织列表v2

> 接口说明
> 根据不同的组织属性分页查询组织信息。
> 查询组织列表接口可以根据组织唯一标识集、组织名称、组织状态这些查询条件来进行高级查询；若不指定查询条件，即全量获取所有的组织信息。返回结果分页展示。
> 注：若指定多个查询条件，表示将这些查询条件进行“与”的组合后进行精确查询。
> 根据“组织名称orgName”查询为模糊查询。
> 根据父组织查询子孙组织，为便于构建组织树，会返回没有权限的父组织，通过available为false区分； 如果额外指定了其它字段，为在返回数据的基础上进一步过滤出符合条件的数据。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/resource/v2/org/advance/orgList`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v2/org/advance/orgList
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orgName`|`body`|`string`|否|组织名称，如默认部门|
|`orgIndexCodes`|`body`|`string`|否|组织唯一标识码集合|
|`pageNo`|`body`|`integer`|是|当前页码|
|`pageSize`|`body`|`integer`|是|实际每页记录总数|
|`parentOrgIndexCodes`|`body`|`string`|否|父组织唯一标示集合，多个值使用英文逗号分隔，不超过1000个|
|`isSubOrg`|`body`|`boolean`|否|true时，搜索parentOrgIndexCodes的所有子孙组织；|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|否|资源属性名，key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|否|操作运算符， 0 ：=， 1 ：>=， 2 ：<=， 3 ：in， 4 ：not in， 5 ：between， 6 ：like， 7 ：pre like， 8 ：suffix like|
|`+values`|`Body`|`string[]`|否|资源属性值，=、>=、<=、like、values数组长度只能是1； in、not in，values数组长度大于1，最大不超时20； in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到； between只能用于整形、日期 ； like只能用于字符串。|
|`orderBy`|`body`|`string`|否|排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序：desc|

补充说明:
- 多个值使用英文逗号分隔，不超过1000个，从获取组织列表接口获取返回参数orgIndexCode
- 当前页码需大于0（pageNo>0）
- 每页记录展示的数目应大于0，小于等于1000（0<pageSize<=1000）
- false时，只搜索直接子组织；
- 升序：asc

## 请求参数示例
```json
{
  "orgName": "x",
  "orgIndexCodes": "fb6a060e-48c4-4348-834c-3755cc699476,18a81b20-41a0-4794-925a-d7a4bcfc7705",
  "pageNo": 1,
  "pageSize": 1000,
  "parentOrgIndexCodes": "asd7d8y89yd812001",
  "isSubOrg": true,
  "expressions": [
    {
      "key": "orgIndexCode",
      "operator": 0,
      "values": [
        "ayd8y80y1y082ye01y2e8"
      ]
    }
  ],
  "orderBy": "orgName",
  "orderType": "asc"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|组织信息结构体|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|每页记录总数|
|`+list`|`body`|`object[]`|否|组织列表|
|`++orgIndexCode`|`body`|`string`|否|组织唯一标识码|
|`++organizationCode`|`body`|`string`|否|组织编号|
|`++orgName`|`body`|`string`|否|组织名称，如默认部门|
|`++orgPath`|`body`|`string`|否|@10935e1d-9a84-46ff-aa17-cdd239ad7e0b@|
|`++parentOrgIndexCode`|`body`|`string`|否|父组织唯一标识码|
|`++available`|`body`|`boolean`|否|用于标识组织节点是否有权限操作|
|`++leaf`|`body`|`boolean`|否|标识组织节点是否叶子节点|
|`++sort`|`body`|`number`|否|显示顺序|
|`++createTime`|`body`|`string`|否|创建时间 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00 表示北京时间2017年7月26日21时30分08秒|
|`++updateTime`|`body`|`string`|否|更新时间 要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00 表示北京时间2017年7月26日21时30分08秒|
