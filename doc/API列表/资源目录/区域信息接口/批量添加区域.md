# 批量添加区域

> 接口说明
> 支持区域的批量添加。
> 三方可以自行指定区域的唯一标识，也可以由ISC平台自行生成。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/region/batch/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/region/batch/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`Array[0]`|`body`|`object`|否|待添加的区域列表，单次提交上限为1000条|
|`+clientId`|`Body`|`number`|否|regionIndexCode为空时，regionIndexCode会由平台生成，三方用户需要通过clientid和regionIndexCode进行关联，再通过返回值中的clientId判断哪些成功，哪些失败。所以建议每次接口调用，clientid保持唯一。|
|`+regionIndexCode`|`Body`|`string`|否|区域唯一标志，不允许与其它区域唯一标志重复，包括已删除的区域，值为空或者不传此字段时系统自动生成唯一标志|
|`+parentIndexCode`|`Body`|`string`|是|父区域唯一标识码，根据查询区域列表v2接口获取返回报文中的indexCode字段|
|`+regionName`|`Body`|`string`|是|区域名称，同一级不可重复|
|`+regionCode`|`Body`|`string`|否|外码编码，regionType为12时有效，regionType为10时忽略该值|
|`+regionType`|`Body`|`number`|是|区域类型，10-普通区域，12-楼栋单元|
|`+description`|`Body`|`string`|否|区域描述信息|

补充说明:
- 注：clientid只对本次调用有效

## 请求参数示例
```json
[
  {
    "clientId": 11,
    "parentIndexCode": "24tvfv3v5v389c8y54c808y5c",
    "regionName": "区域1",
    "regionCode": "3q8c8y820y",
    "regionType": 10,
    "description": "区域描述"
  }
]
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回结果|
|`+successes`|`body`|`object[]`|否|批量添加成功的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++regionIndexCode`|`body`|`string`|否|服务端生成的唯一标识|
|`+failures`|`body`|`object[]`|否|批量添加失败的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++code`|`body`|`string`|否|错误码|
|`++msg`|`body`|`string`|否|错误信息|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "successes": [
      {
        "clientId": 123456789,
        "regionIndexCode": "5668f44c-ae23-4674-9b79-d8f79f77fd41"
      }
    ],
    "failures": [
      {
        "clientId": 555,
        "code": "-1",
        "msg": "error"
      }
    ]
  }
}
```
