# 查询区域列表v2

> 接口说明
> 根据查询条件查询区域列表信息，主要用于区域信息查询过滤。
> 相对V1接口，支持级联场景的区域查询

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/irds/v2/region/nodesByParams`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/irds/v2/region/nodesByParams
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`resourceType`|`body`|`string`|是|资源类型;详见数据字典：|
|`parentIndexCodes`|`body`|`string[]`|否|父编号集合;|
|`isSubRegion`|`body`|`boolean`|否|是否包含下级区域，true时，搜索parentIndexCodes的所有子、孙区域；|
|`pageNo`|`body`|`integer`|是|当前页码；pageNo≥1|
|`pageSize`|`body`|`integer`|是|分页大小；0<pageSize≤1000|
|`authCodes`|`body`|`string[]`|否|权限码集合;|
|`regionType`|`body`|`integer`|否|区域类型，10-普通区域，11-级联区域，12-楼栋单元|
|`regionName`|`body`|`string`|否|区域名称；根据区域名称过滤，可模糊查询；|
|`sonOrgIndexCodes`|`body`|`string[]`|否|本级区域向上查询;|
|`cascadeFlag`|`body`|`integer`|否|级联标识，|
|`orderBy`|`body`|`string`|否|排序字段,注意：排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序升序,降序：desc，|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|否|资源属性名，key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|否|操作运算符， 0 ：=， 1 ：>=， 2 ：<=， 3 ：in， 4 ：not in， 5 ：between， 6 ：like， 7 ：pre like， 8 ：suffix like|
|`+values`|`Body`|`string[]`|否|资源属性值，=、>=、<=、like、values数组长度只能是1； in、not in，values数组长度大于1，最大不超时20； in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到； between只能用于整形、日期 ； like只能用于字符串。|

补充说明:
- 附录A.3 资源类型/资源权限码；
- 查询资源的类型，传region时查询的为用户有配置权限的区域树，传资源类型如：camera、encodeDevice查询用户对该类资源有权限的区域树；
- 注：资源iasDevice\reader\floor不进行权限校验，即不传authCodes
- String[]；
- 父编号个数
- <=1000个；
- 单个长度<=64Byte；
- false时，只搜索parentIndexCodes的直接子区域；
- String[]；详见数据字典
- 附录A.3 资源类型/资源权限码，
- 权限码个数
- <=20个；
- 单个权限码长度
- <=40Byte；
- 只有同时指定userId、authCodes时，才进行权限过滤；
- 当指定多个权限码时，只返回同时具有这些权限码的区域；
- 若包含中文，最大长度40，最大长度指不超过按照指定编码的字节长度，即getBytes(“utf-8”).length
- String[];场景：通过名称模糊查询资源点获取区域编号，并组成资源点的树形结构；
- 本级区域个数
- <=10个；
- 单个长度<=64Byte；
- sonOrgIndexCodes集合中的多个区域只能为同一级目录；
- 当parentIndexCodes、regionType、regionName都为空时，才可根据sonOrgIndexCodes查询；
- 0-全部，
- 1-本级，
- 2-级联，
- 默认0
- 升序：asc

## 请求参数示例
```json
{
  "resourceType": "region",
  "parentIndexCodes": [
    "root000000"
  ],
  "isSubRegion": true,
  "pageNo": 1,
  "pageSize": 1,
  "authCodes": [
    "view"
  ],
  "regionType": 10,
  "regionName": "区",
  "sonOrgIndexCodes": [
    "root000000"
  ],
  "cascadeFlag": 0,
  "orderBy": "regionName",
  "orderType": "desc",
  "expressions": [
    {
      "key": "updateTime",
      "operator": 5,
      "values": [
        "2019-07-24T11:57:02.792+08:00",
        "2019-07-25T11:57:02.792+08:00"
      ]
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|区域信息结构体|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码范围 ( 0 , ~ )，最大为整数最大值，若是范围内数字超过实际最大页码值，返回最后一页记录 注：0 < pageNo|
|`+pageSize`|`body`|`number`|否|每页记录总数 范围 ( 0 , 1000 ] 注：0 < pageSize≤1000|
|`+list`|`body`|`object[]`|否|对象列表|
|`++indexCode`|`body`|`string`|否|区域编号|
|`++name`|`body`|`string`|否|区域名称|
|`++regionPath`|`body`|`string`|否|区域完整目录，含本节点，/进行分割，上级节点在前|
|`++parentIndexCode`|`body`|`string`|否|父区域唯一标识码|
|`++available`|`body`|`boolean`|否|用于标识区域节点是否有权限操作，true：有权限 false：无权限|
|`++leaf`|`body`|`boolean`|否|true:是叶子节点，表示该区域下面未挂区域 false:不是叶子节点，表示该区域下面挂有区域|
|`++cascadeCode`|`body`|`string`|否|级联平台标识，多个级联编号以@分隔，本级区域默认值“0”|
|`++cascadeType`|`body`|`number`|否|区域标识 0：本级 1：级联 2：混合，下级推送给上级的本级点（杭州本级有滨江，然后下级滨江又把自己推送上来了，滨江是混合区域节点） 入参cascadeFlag与返回值对应： cascadeFlag=0：返回0、1、2 cascadeFlag=1：返回0、2 cascadeFlag=2：返回1、2|
|`++catalogType`|`body`|`number`|否|区域类型， 国标区域：0， 雪亮工程区域：1， 司法行政区域：2， 自定义区域：9， 历史兼容版本占用普通区域:10， 历史兼容版本占用级联区域:11， 楼栋单元:12|
|`++externalIndexCode`|`body`|`string`|否|外码(如：国际码)|
|`++parentExternalIndexCode`|`body`|`string`|否|父外码(如：国际码)|
|`++sort`|`body`|`number`|否|同级区域顺序|
|`++localQuantity`|`body`|`number`|否|本区域资源数量(根据资源类型统计)，只统计本级挂的资源数量，不包含下级及下下级等|
|`++totalQuantity`|`body`|`number`|否|本区域及下级区域资源数量，包含本级及下级|
|`++createTime`|`body`|`string`|否|创建时间，要求遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|
|`++updateTime`|`body`|`string`|否|创建时间，要求遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|
