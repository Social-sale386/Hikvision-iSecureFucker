# 根据区域编号获取下一级区域列表v2

> 接口说明
> 根据用户请求的资源类型和资源权限获取父区域的下级区域列表，主要用于逐层获取父区域的下级区域信息，例如监控点预览业务的区域树的逐层获取。下级区域只包括直接下级子区域。
> 注：
> 查询区域管理权限（resourceType为region），若父区域的子区域无权限、但是其孙区域有权限时，会返回该无权限的子区域，但是该区域的available标记为false（表示无权限）

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v2/regions/subRegions`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v2/regions/subRegions
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`parentIndexCode`|`body`|`string`|是|父区域编号，-1为顶节点的父区域编号；|
|`resourceType`|`body`|`string`|是|资源类型，详见附录A.3 资源类型/资源权限码 ，|
|`authCodes`|`body`|`array`|否|权限码集合;|
|`pageNo`|`body`|`integer`|是|当前页码；pageNo≥1|
|`pageSize`|`body`|`integer`|是|分页大小；0<pageSize≤1000|
|`cascadeFlag`|`body`|`integer`|否|级联标识，|

补充说明:
- 即当指定parentIndexCode为-1时，表示查询根节点信息；根据查询区域列表v2接口获取返回报文中的indexCode字段
- 查询资源的类型，传region时查询的为用户有配置权限的区域树，传资源类型如：camera、encodeDevice查询用户对该类资源有权限的区域树；
- 注：资源iasDevice\reader\floor不进行权限校验，即不传authCodes
- String[]；，详见附录A.3 资源类型/资源权限码，
- 权限码个数
- <=20个；
- 单个权限码长度
- <=40Byte；
- 只有同时指定userId、authCodes时，才进行权限过滤；
- 当指定多个权限码时，只返回同时具有这些权限码的区域
- 0-全部，
- 1-本级，
- 2-级联，
- 默认0

## 请求参数示例
```json
{
  "parentIndexCode": "root000000",
  "resourceType": "camera",
  "pageNo": 1,
  "pageSize": 1,
  "cascadeFlag": 0
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|区域信息结构体|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码范围 ( 0 , ~ )，最大为整数最大值，若是范围内数字超过实际最大页码值，返回最后一页记录 注：0 < pageNo|
|`+pageSize`|`body`|`number`|否|每页记录总数 范围 ( 0 , 1000 ] 注：0 < pageSize≤1000|
|`+list`|`body`|`object[]`|否|对象列表|
|`++indexCode`|`body`|`string`|否|区域唯一标识码|
|`++name`|`body`|`string`|否|区域名称|
|`++parentIndexCode`|`body`|`string`|否|父区域唯一标识码|
|`++available`|`body`|`boolean`|否|用于标识区域节点是否有权限操作，true：有权限 false：无权限|
|`++leaf`|`body`|`boolean`|否|true:是叶子节点，表示该区域下面未挂区域 false:不是叶子节点，表示该区域下面挂有区域|
|`++cascadeCode`|`body`|`string`|否|级联平台标识，多个级联编号以@分隔，本级区域默认值“0”|
|`++cascadeType`|`body`|`number`|否|区域标识 0：本级 1：级联 2：混合，下级推送给上级的本级点（杭州本级有滨江，然后下级滨江又把自己推送上来了，滨江是混合区域节点） 入参cascadeFlag与返回值对应： cascadeFlag=0：返回0、1、2 cascadeFlag=1：返回0、2 cascadeFlag=2：返回1、2|
|`++catalogType`|`body`|`number`|否|区域类型， 国标区域：0， 雪亮工程区域：1， 司法行政区域：2， 自定义区域：9， 历史兼容版本占用普通区域:10， 级联区域:11， 楼栋单元:12|
|`++externalIndexCode`|`body`|`string`|否|外码(如：国际码)|
|`++sort`|`body`|`number`|否|同级区域顺序|
|`++regionPath`|`body`|`string`|否|区域完整目录，含本节点，@进行分割，上级节点在前|
|`++createTime`|`body`|`string`|否|创建时间，要求遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|
|`++updateTime`|`body`|`string`|否|创建时间，要求遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|
