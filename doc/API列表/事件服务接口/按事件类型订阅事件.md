# 按事件类型订阅事件

> 接口说明
> 该接口用于满足应用方按事件类型码订阅事件，同一个用户重复订阅相同的事件，接口内部逻辑自动去重，确保不重复投递事件。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/eventService/v1/eventSubscriptionByEventTypes`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/eventService/v1/eventSubscriptionByEventTypes
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`eventTypes`|`body`|`integer[]`|是|事件类型,参考附录D 事件列表|
|`eventDest`|`body`|`string`|是|指定事件接收的地址，采用restful回调模式，支持http和https，样式如下：http://ip:port/eventRcv或者|
|`subType`|`body`|`integer`|否|订阅类型 0：订阅原始事件 1：联动事件 2：原始事件和联动事件，不填使用默认值0|
|`eventLvl`|`body`|`integer[]`|否|事件等级，订阅类型为0时，此参数无效，使用默认值0|

补充说明:
- https://ip:port/eventRcv
- 不超过1024个字符，事件接收地址由应用方负责按指定的规范提供，报文格式如附录D.3事件报文格式，事件接收接口不需要认证。
- 注意：三方客户收到消息, 请注意立即返回HTTP/1.1 200 OK, 否则因为接收太慢,导致事件积压
- 在订阅类型为1时，不填使用默认值[1,2,3]
- 在订阅类型为2时，不填使用默认值[0,1,2,3]
- 数组大小不超过32，事件等级大小不超过31

## 请求参数示例
```json
{
  "eventTypes": [
    131329
  ],
  "eventDest": "https://ip:port/eventRcv",
  "subType": 1,
  "eventLvl": [
    2
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，|
|`msg`|`body`|`string`|否|返回描述-记录接口执行情况说明信息|

补充说明:
- 0表示订阅成功，其它参考附录E.other.2 事件订阅返回码说明
- success表示成功描述，其他的如附录E.2 事件订阅返回码说明

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success"
}
```
