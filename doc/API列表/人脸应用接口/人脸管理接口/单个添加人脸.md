# 单个添加人脸

> 接口说明
> a)一次性添加一张人脸，返回结果为添加成功的人脸。
> b)添加的人脸图片，目前支持URL方式和二进制数据方式。URL方式时，智能监控应用服务会通过HTTP协议的GET方式下载图片，校验图片，最后重新上传图片；二进制数据是指图片的字节流经过Base64编码后得到的字符串。
> c)该URL能通过HTTP协议的GET方式能下载获取到即可，若包含认证，由调用方在URL中加上认证信息，保证URL能成功访问。
> d)智能监控应用服务会根据图片存储位置配置，将图片上传到图片存储服务器中，返回的URL为图片存储服务器上的相对地址。
> e)图片大小和格式均有要求，上传大小在10KB到200KB间的图片，上传JGP格式的图片。
> f)若添加的人脸对应的人脸分组已经配置有识别计划，则新添加的人脸会被一并下发到设备上。
> g)该接口依赖于图片存储位置已配置完，请确保平台已经配置有人脸图片存储位置，否则添加必定失败。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/face/single/addition`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/face/single/addition
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`faceGroupIndexCode`|`body`|`string`|是|指定人脸添加到的人脸分组的唯一标识,智能监控应用服务会校验该值的有效性，可从按条件查询人脸分组 接口返回报文中的indexCode|
|`faceInfo`|`body`|`object`|是|人脸信息对象|
|`+name`|`Body`|`string`|是|人脸的名称,1~32个字符；不能包含 ’ / \ : * ? " < >|
|`+certificateType`|`Body`|`string`|否|人脸的证件类别，111-身份证，OTHER-其它证件|
|`+certificateNum`|`Body`|`string`|否|人脸的证件号码信息。1~20个数字、字母。|
|`facePic`|`body`|`object`|是|人脸图片对象|
|`+faceUrl`|`Body`|`string`|否|人脸图片的URL。和faceBinaryData不能同时为空，同时存在时优先取faceBinaryData；图片的大小范围在10KB到200KB之间，只支持JGP格式图片。|
|`+faceBinaryData`|`Body`|`string`|否|人脸图片的二进制数据经过Base64编码后的字符串，和faceUrl不能同时为空，同时存在时优先取faceBinaryData。 图片的大小范围在10KB到200KB之间，只支持JGP格式图片。|

补充说明:
- 最大长度：200*1024

## 请求参数示例
```json
{
  "faceGroupIndexCode": "7cc0aCC2-a3c3-48fd-b432-718103e85c28",
  "faceInfo": {
    "name": "张三",
    "certificateType": "111",
    "certificateNum": "420204199605121656"
  },
  "facePic": {
    "faceUrl": "http://10.166.165.121:8080/frs/facepicturetemp/test.JGP"
  }
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|添加成功的人脸|
|`+indexCode`|`body`|`string`|否|人脸的唯一标识|
|`+faceGroupIndexCode`|`body`|`string`|否|人脸所属的人脸分组的唯一标识|
|`+faceInfo`|`body`|`object`|否|人脸信息|
|`++name`|`body`|`string`|否|人脸的名称,1~32个字符；不能包含 ’ / \ : * ? " < >|
|`++certificateType`|`body`|`string`|否|人脸的证件类别，111-身份证，OTHER-其它证件|
|`++certificateNum`|`body`|`string`|否|人脸的证件号码信息。1~20个数字、字母|
|`+facePic`|`body`|`object`|否|人脸的图片信息|
|`++faceUrl`|`body`|`string`|否|人脸图片的URL。和faceBinaryData不能同时为空，同时存在时优先取faceBinaryData|
