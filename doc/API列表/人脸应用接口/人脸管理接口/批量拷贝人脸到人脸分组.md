# 批量拷贝人脸到人脸分组

> 接口说明
> a)批量将若干人脸拷贝到若干分组内。
> b)这个接口可以实现将平台已经存在的人脸迁移到其它分组内。
> c)无论如何调用该接口，同一个分组内不会出现两个indexCode相同的人脸。
> d)若人脸已经在人脸分组内了，重复的迁移操作也会返回成功。
> e)拷贝一次性最多拷贝1000张图片，最多同时拷贝到16个分组内。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/face/batch/copy`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/face/batch/copy
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCodes`|`body`|`string[]`|是|待拷贝的若干人脸唯一标识集合，可从按条件批量查询人脸接口返回报文中的indexCode字段|
|`faceGroupIndexCodes`|`body`|`string[]`|是|目标人脸分组的唯一标识集合，可从按条件查询人脸分组接口返回报文中的indexCode字段|

## 请求参数示例
```json
{
  "indexCodes": [
    "5dc82633-a4cb-4107-b55e-f21bbdf952f9"
  ],
  "faceGroupIndexCodes": [
    "5dc82633-a4cb-4107-b55e-f21bbdf952f5"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|拷贝是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
