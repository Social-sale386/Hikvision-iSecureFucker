# 查询陌生人识别计划

> 接口说明
> a)根据查询条件，查询符合条件的陌生人识别计划的集合。
> b)查询条件之间的关系为与，即所有条件同时生效，不传任何条件代表查询所有。
> c)这里的人脸分组集合作为查询条件，如果传多个分组，则识别计划只要包含其中任意一个分组，就会被查询出来。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/recognition/white`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/recognition/white
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`description`|`body`|`string`|否|根据识别计划描述模糊搜索|
|`faceGroupIndexCodes`|`body`|`string[]`|否|根据人脸分组唯一标识集合搜索|
|`name`|`body`|`string`|否|根据识别计划名称模糊搜索|
|`status`|`body`|`string`|否|根据下发状态搜索，详见[附录A.33 识别计划状态类型](/docs/docId?productId=5c67f1e2f05948198c909700&version=/29c78ef52ca842c7933bd2b8e051e9d0&tagPath=附录-附录A.33 识别计划状态类型)|

## 请求参数示例
```json
{
  "description": "这是一个描述",
  "faceGroupIndexCodes": [
    "string"
  ],
  "name": "仓库值守陌生人识别计划",
  "status": "RUNNING"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object[]`|否|查询到的识别计划集合|
|`+indexCode`|`body`|`string`|否|识别计划的唯一标识|
|`+name`|`body`|`string`|否|识别计划的名称|
|`+faceGroupIndexCodes`|`body`|`string[]`|否|人脸分组的唯一标识集合|
|`+cameraIndexCodes`|`body`|`string[]`|否|抓拍点的唯一标识集合|
|`+recognitionResourceIndexCodes`|`body`|`string[]`|否|识别资源的唯一标识集合|
|`+recognitionResourceType`|`body`|`string`|否|识别资源类型，SUPER_BRAIN 超脑，FACE_RECOGNITION_SERVER 脸谱，COMPARISON 海康深眸|
|`+recognitionPlanType`|`body`|`string`|否|识别计划的类型, BLACK_LIST-重点人员识别计划 WHITE_LIST-陌生人识别计划|
|`+description`|`body`|`string`|否|识别计划的描述|
|`+status`|`body`|`string`|否|识别计划的状态，详见[附录A.33 识别计划状态类型](/docs/docId?productId=5c67f1e2f05948198c909700&version=/29c78ef52ca842c7933bd2b8e051e9d0&tagPath=附录-附录A.33 识别计划状态类型)|
|`+available`|`body`|`boolean`|否|是否启用,true:启用,false:禁用|
|`+startTime`|`body`|`string`|否|识别计划开始下发的时间|
|`+usingTime`|`body`|`number`|否|识别计划下发已经用的时间|
|`+threshold`|`body`|`number`|否|阈值，范围[1,100)|
