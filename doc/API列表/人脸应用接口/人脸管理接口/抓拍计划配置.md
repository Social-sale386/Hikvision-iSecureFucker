# 抓拍计划配置

> 接口说明
> a)配置平台的抓拍计划，目前支持开启和关闭两种操作。
> b)当抓拍计划关闭后，也就无法从消息队列中获取到人脸事件了。
> c)关闭抓拍计划不会影响到识别计划，因此人脸比对报警、重点人员识别报警、陌生人识别报警，仍旧会上报。
> d)关闭抓拍计划的接口，是为了在部分场景下，去除无用的抓拍事件。这样可以释放消息队列、事件服务和智能监控的压力，将性能用在更需要的地方。
> e)重复的开启或者关闭，接口会返回成功。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/capture/configuration`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/capture/configuration
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`eventSwitch`|`body`|`boolean`|是|开启或者关闭平台的抓拍计划，true:开启|

补充说明:
- false:关闭

## 请求参数示例
```json
{
  "eventSwitch": true
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|删除是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
