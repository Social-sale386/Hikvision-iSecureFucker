# 删除人脸分组

> 接口说明
> a)根据删除条件，删除一批人脸分组,最大1000个，返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)已经配置有识别计划的人脸分组，不允许删除，由特定错误码标识。
> c)根据唯一标识删除时，会校验唯一标识的有效性，若任意分组唯一标识无效，则返回错误。传入的唯一标识重复，做去重处理。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/face/group/batch/deletion`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/face/group/batch/deletion
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCodes`|`body`|`string[]`|是|根据分组的唯一标识删除|

## 请求参数示例
```json
{
  "indexCodes": [
    "5dc82633-a4cb-4107-b55e-f21bbdf952f9"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述,描述错误码的原因|
|`data`|`body`|`boolean`|否|删除是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success",
  "data": true
}
```
