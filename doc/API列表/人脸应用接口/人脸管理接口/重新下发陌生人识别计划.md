# 重新下发陌生人识别计划

> 接口说明
> a)重新下发指定的陌生人识别计划，返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)重新下发只能针对状态为下发失败的识别计划操作，且重新下发只会下发失败的部分，对于已经成功的部分不会重复下发。
> c)重新下发的结果可以通过接口”单个查询陌生人识别计划详情”查看。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/recognition/white/restart`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/recognition/white/restart
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCode`|`body`|`string`|是|识别计划的唯一标识，可从查询陌生人识别计划接口返回报文中的indexCode字段|

## 请求参数示例
```json
{
  "indexCode": "7cc0aCC2-a3c3-48fd-b432-718103e85c28"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|重新下发是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
