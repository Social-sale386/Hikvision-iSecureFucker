# 单个修改人脸

> 接口说明
> a)修改单张人脸信息,返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)若修改的人脸对应的人脸分组已经配置有识别计划，则修改后的人脸会被重新下发到设备上。
> c)修改后的字段信息以修改时传入的字段为准，若不传字段或传入null字段，则该字段会被置为null。
> d)如果要修改人脸信息，则需要传递faceInfo对象。
> e)如果要修改人脸图片，则需要传递facePic对象。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/face/single/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/face/single/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCode`|`body`|`string`|是|人脸的唯一标识，可从按条件批量查询人脸接口返回报文中的indexCode字段|
|`faceInfo`|`body`|`object`|否|人脸信息|
|`+name`|`Body`|`string`|是|人脸的名称,1~32个字符；不能包含 ’ / \ : * ? " < >|
|`+certificateType`|`Body`|`string`|否|人脸的证件类别信息|
|`+certificateNum`|`Body`|`string`|否|人脸的证件号码信息。1~20个数字、字母|
|`facePic`|`body`|`object`|否|人脸图片|
|`+faceBinaryData`|`Body`|`string`|否|人脸图片的二进制数据经过Base64编码后的字符串。和faceUrl不能同时为空，同时存在时优先取faceBinaryData，图片的大小范围在10KB到200KB之间，只支持JGP格式图片。|
|`+faceUrl`|`Body`|`string`|否|人脸图片的URL。和faceBinaryData不能同时为空，同时存在时优先取faceBinaryData，图片的大小范围在10KB到200KB之间，只支持JGP格式图片。|

## 请求参数示例
```json
{
  "indexCode": "7cc0aCC2-a3c3-48fd-b432-718103e85c28",
  "faceInfo": {
    "name": "张三",
    "certificateType": "111",
    "certificateNum": "420204199605121656"
  },
  "facePic": {
    "faceUrl": "http://10.166.165.121:8080/frs/facepicturetemp/test.JGP"
  }
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|修改是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
