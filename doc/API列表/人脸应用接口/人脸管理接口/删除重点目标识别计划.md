# 删除重点目标识别计划

> 接口说明
> a)根据重点目标识别计划的唯一标识,删除若干重点人员识别计划，返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)删除识别计划后，会异步删除识别计划中下发到设备上的人脸分组、人脸、布防信息。因此调用接口成功的时候，可以观察到识别资源上的分组还是存在的，需要等一段时间后才会删除。
> c)请勿在识别计划正在下发时，调用删除识别计划的接口。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/recognition/black/deletion`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/recognition/black/deletion
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCodes`|`body`|`string[]`|是|操作的计划的唯一标识集合|

## 请求参数示例
```json
{
  "indexCodes": [
    "7cc0aCC2-a3c3-48fd-b432-718103e85c28"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|删除是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
