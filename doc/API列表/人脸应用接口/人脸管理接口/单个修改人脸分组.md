# 单个修改人脸分组

> 接口说明
> a)修改单个人脸分组，返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)修改后的字段信息以修改时传入的字段为准，若不传字段或传入null字段，则该字段会被置为null。
> c)若该人脸分组已经被配置到识别计划中，则修改人脸分组后，会将修改后的信息，一并下发到设备上。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/face/group/single/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/face/group/single/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCode`|`body`|`string`|是|人脸分组的唯一标识，可从按条件查询人脸分组接口返回报文中的indexCode字段|
|`name`|`body`|`string`|是|人脸分组的名称，1~32个字符；不能包含 ’ / \ : * ? " < >|
|`description`|`body`|`string`|是|分组的描述，1~128个字符|

## 请求参数示例
```json
{
  "indexCode": "5dc82633-a4cb-4107-b55e-f21bbdf952f9",
  "name": "仓库值守人员",
  "description": "仓库值守人员是指守着仓库的人"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|修改是否成功,true成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success",
  "data": true
}
```
