# 获取人脸及分组关联关系

> 接口说明
> a)已经配置了一条或多条识别计划。
> b)调用方获取到人脸比对事件后，可通过调用该接口获取关联关系，将人脸比对事件报文中的人脸编号、分组编号和智能监控应用服务中相对应。
> c)调用方可在识别计划下发完成后调用该接口获取关联关系。
> d)严禁在事件接收时调用此接口，避免造成接口性能不佳。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/getFaceRelation`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/getFaceRelation
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCodes`|`body`|`string[]`|是|人脸唯一编号数组，可从按条件批量查询人脸接口返回报文中的indexCode字段|

## 请求参数示例
```json
{
  "indexCodes": [
    "3045a9d0-f2c8-469f-ab01-934a6CC26cd6"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object[]`|否|查询到的人脸关联关系对象数组|
|`+faceIndexCode`|`body`|`string`|否|智能监控应用服务人脸编号|
|`+faceGroupIndexCode`|`body`|`string`|否|智能监控应用服务人脸分组唯一编号|
|`+resFaceIndexCode`|`body`|`string`|否|识别资源上人脸的唯一编号|
|`+resFaceGroupIndexCode`|`body`|`string`|否|识别资源上人脸库的唯一编号|
|`+resIndexCode`|`body`|`string`|否|识别资源的唯一编号|

```json
    }

```
