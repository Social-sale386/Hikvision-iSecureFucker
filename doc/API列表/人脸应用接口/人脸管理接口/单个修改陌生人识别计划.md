# 单个修改陌生人识别计划

> 接口说明
> a)修改指定的陌生人识别计划信息，返回的data为布尔类型，true代表操作成功，false代表操作失败。
> b)修改识别计划时，传入的信息和添加一致，不需要修改的字段也要传过来，但是部分不支持修改的字段不需要传，详情请以入参说明为准。
> c)修改识别计划的接口校验同添加一致，请见单个添加陌生人识别计划中的接口说明，这里不再赘述。
> d)识别计划修改后，会自动重新下发。
> e)海康深眸、超脑作为识别资源时不支持修改识别资源的类型。
> f) 识别计划支持配置时间计划模板，当时间计划参数不传时，默认全天候。当传入此参数时，时间计划规则如下：按周计划进行配置，每一天的时间段上限为8，每一个时间段的开始时间必须小于结束时间。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/frs/v1/plan/recognition/white/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/frs/v1/plan/recognition/white/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`indexCode`|`body`|`string`|是|识别计划唯一标识，可从查询陌生人识别计划接口返回报文中的indexCode字段|
|`name`|`body`|`string`|是|识别计划名称,1~32个字符；不能包含 ’ / \ : * ? " < >|
|`faceGroupIndexCodes`|`body`|`string[]`|是|人脸分组的唯一标示，可从按条件查询人脸分组接口返回报文中的indexCode字段|
|`recognitionResourceIndexCodes`|`body`|`string[]`|否|识别资源唯一标识的集合。智能监控应用服务会校验识别资源标识的有效性，若有重复、无效的标识，校验不通过。识别资源类型为海康深眸时，可以为空。可从按条件查询识别资源接口返回报文中的indexCode字段|
|`cameraIndexCodes`|`body`|`string[]`|是|监控点唯一标识，可从分页获取监控点资源接口返回报文中的indexCode字段|
|`description`|`body`|`string`|否|识别计划描述，1~128个字符|
|`threshold`|`body`|`integer`|是|识别计划的阈值，陌生人|
|`timeBlockList`|`body`|`object[]`|否|时间计划模板|
|`+dayOfWeek`|`Body`|`string`|是|星期几，1-7分别表示周一至周七|
|`+timeRange`|`Body`|`object[]`|是|时间段数组，一天可以有多个时间段|
|`++startTime`|`Body`|`string`|是|开始时间，时分，如“00:00”|
|`++endTime`|`Body`|`string`|是|结束时间，时分，如“00:00”|

补充说明:
- 是指相似度不足该值时报
- 警 ，范围 [1,100)

## 请求参数示例
```json
{
  "indexCode": "7cc0aCC2-a3c3-48fd-b432-718103e85c28",
  "name": "仓库值守陌生人识别计划",
  "faceGroupIndexCodes": [
    "8ce19be05b5c4914b1e3c9eee3997eb2"
  ],
  "recognitionResourceIndexCodes": [
    "867c73b5-56d8-43b7-8491-6481ab23c8be"
  ],
  "cameraIndexCodes": [
    "c4023a78c83948f9b6b0fb76b0f3CC71"
  ],
  "description": "这是一个描述",
  "threshold": 32,
  "timeBlockList": [
    {
      "dayOfWeek": "1",
      "timeRange": [
        {
          "startTime": "00:00",
          "endTime": "12:00"
        }
      ]
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.5 智能监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`boolean`|否|修改是否成功,true：成功,false：失败|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "Success.",
  "data": true
}
```
