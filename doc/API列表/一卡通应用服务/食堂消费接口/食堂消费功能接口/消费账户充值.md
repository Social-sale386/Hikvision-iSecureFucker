# 消费账户充值

> 接口说明
> 该接口可以通过账户类型、卡号或人员唯一ID对人员普通/补贴账户进行充值。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/cems/v1/account/recharge`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/cems/v1/account/recharge
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personId`|`body`|`string`|否|人员唯一ID，如为空则cardNo不允许为空，如果人员Id和卡号同时不为空，按人员Id来充值，根据查询人员列表v2接口获取返回报文中的personId字段|
|`cardNo`|`body`|`string`|否|卡号,由刷卡终端设备触发读取的卡号，如为空则personId不允许为空，根据获取卡片列表接口获取返回报文中的cardId字段|
|`money`|`body`|`integer`|是|充值金额,单位:分,范围0 < money < 10000000|
|`accountType`|`body`|`integer`|是|账户类型,账户类型:|
|`rechargeId`|`body`|`string`|是|充值唯一编号，重复编号只充值一次|

补充说明:
- 1.普通账户
- 2.补贴账户

## 请求参数示例
```json
{
  "personId": "h18oqjdfv80148ydx8h8y58180",
  "cardNo": "1234567864",
  "money": 2000,
  "accountType": 1,
  "rechargeId": "abcdefg-hijklmn-opq-rst"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.6 食堂消费错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+personName`|`body`|`string`|否|账户所属人员名称|
|`+genAccount`|`body`|`number`|否|普通账户金额，单位：分|
|`+money`|`body`|`number`|否|充值金额,单位:分,范围0 < money < 10000000|
|`+subAccount`|`body`|`number`|否|补贴账户金额，单位：分|
|`+totalAccount`|`body`|`number`|否|账户总金额，单位：分|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "personName": "张三",
    "genAccount": 100,
    "money": 100,
    "subAccount": 200,
    "totalAccount": 300
  }
}
```
