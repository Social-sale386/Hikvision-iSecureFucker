# 查询账户记录

> 接口说明
> 该接口可以查询人员账户的记录，支持按照人员唯一标示、交易类型、时间段查询人员账户记录信息

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/cems/v1/account/log/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/cems/v1/account/log/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`startTime`|`body`|`string`|是|记录开始时间，ISO8601格式:|
|`endTime`|`body`|`string`|是|记录结束时间，ISO8601格式:|
|`transactionTypes`|`body`|`string`|否|账户交易类型： 1：充值 2：退款 3：补贴 4：纠正 5：消费扣款 6：补贴账户清空，多个用“,”隔开|
|`personId`|`body`|`string`|否|人员唯一ID，根据查询人员列表v2接口获取返回报文中的personId字段|
|`cardNo`|`body`|`string`|否|卡号，根据获取卡片列表接口获取返回报文中的cardId字段|
|`pageNo`|`body`|`integer`|是|当前页码，pageNo>0|
|`pageSize`|`body`|`integer`|是|每页记录数，0<pageSize<=1000|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，例如北京时间：
- 2018-07-26T15:00:00 + 08:00
- 开始时间和结束时间最大跨度控制在30天
- yyyy-MM-ddTHH:mm:ss+当前时区，例如北京时间：
- 2018-07-26T15:00:00 + 08:00
- 开始时间和结束时间最大跨度控制在30天

## 请求参数示例
```json
{
  "startTime": "2018-07-26T15:00:00+08:00",
  "endTime": "2018-07-26T15:00:00+08:00",
  "transactionTypes": "1,2,3,4,5",
  "personId": "HZ12345",
  "cardNo": "122222333",
  "pageNo": 1,
  "pageSize": 20
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.6 食堂消费错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|查询总数|
|`+totalPage`|`body`|`number`|否|总页数|
|`+page`|`body`|`number`|否|页码|
|`+pageSize`|`body`|`number`|否|页大小|
|`+rows`|`body`|`object[]`|否|返回对象列表|
|`++personId`|`body`|`string`|否|人员唯一ID|
|`++personName`|`body`|`string`|否|人员姓名|
|`++orgName`|`body`|`string`|否|人员所属组织名|
|`++accountType`|`body`|`string`|否|账户类型|
|`++deduction`|`body`|`number`|否|扣款金额，单位：分|
|`++balance`|`body`|`number`|否|当前余额，单位：分|
|`++debitTime`|`body`|`string`|否|交易时间，ISO8601格式：|
|`++transactionType`|`body`|`number`|否|账户交易类型：|
|`++typeName`|`body`|`string`|否|交易类型名称|
|`++remark`|`body`|`string`|否|备注|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，例如北京时间：
- 2018-07-26T15:00:00 + 08:00
- 1：充值
- 2：退款
- 3：补贴
- 4：纠正
- 5：消费扣款
- 6：补贴账户清空
