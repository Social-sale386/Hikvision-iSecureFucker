# 查询消费设备列表

> 接口说明
> 根据条件查询目录下有权限的消费设备列表

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/resource/v1/ccsDevice/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/ccsDevice/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`name`|`body`|`string`|否|名称，模糊搜索，最大长度32，若包含中文，最大长度指不超过按照指定编码的字节长度，即getBytes(“utf-8”).length|
|`regionIndexCodes`|`body`|`string[]`|否|区域编号,可以为空;|
|`isSubRegion`|`body`|`boolean`|否|true时，搜索regionIndexCodes及其子孙区域的资源；|
|`pageNo`|`body`|`integer`|是|当前页码|
|`pageSize`|`body`|`integer`|是|分页大小|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|否|资源属性名，目前支持作为查询条件的字段包括:indexCode、installLocation、updateTime、createTime、resourceType、regionIndexCode、name、description、treatyType。如：key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|否|操作运算符， 0 ：=， 1 ：>=， 2 ：<=， 3 ：in， 4 ：not in， 5 ：between， 6 ：like， 7 ：pre like， 8 ：suffix like|
|`+values`|`Body`|`string[]`|否|资源属性值，=、>=、<=、like、values数组长度只能是1； in、not in，values数组长度大于1，最大不超时20； in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到； between只能用于整形、日期 ； like只能用于字符串。|
|`orderBy`|`body`|`string`|否|排序字段,注意：排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序升序,降序：desc 升序：asc|

补充说明:
- String[]，支持根据区域批量查询；
- 区域编号个数
- <=1000个；
- 单个长度<=64Byte；，可从查询区域列表v2接口获取返回参数indexCode
- false时，只搜索 regionIndexCodes的资源

## 请求参数示例
```json
{
  "name": "test",
  "regionIndexCodes": [
    "8fyc8qw8280y0y43"
  ],
  "isSubRegion": true,
  "pageNo": 1,
  "pageSize": 1,
  "expressions": [
    {
      "key": "updateTime",
      "operator": 5,
      "values": [
        "2019-07-24T11:57:02.792+08:00",
        "2019-07-25T11:57:02.792+08:00"
      ]
    }
  ],
  "orderBy": "name",
  "orderType": "desc"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|分页大小|
|`+list`|`body`|`object[]`|否|资源数据列表|
|`++indexCode`|`body`|`string`|否|唯一编码|
|`++resourceType`|`body`|`string`|否|资源类型，ccsDevice：|
|`++name`|`body`|`string`|否|资源名称|
|`++ip`|`body`|`string`|否|IP|
|`++port`|`body`|`string`|否|端口|
|`++longitude`|`body`|`number`|否|经度，精确到小数点后8位|
|`++latitude`|`body`|`number`|否|纬度，精确到小数点后8位|
|`++elevation`|`body`|`string`|否|海拔高度，单位：米|
|`++capability`|`body`|`string`|否|能力集，详见附录A.44 设备能力集|
|`++deviceKey`|`body`|`string`|否|设备驱动|
|`++deviceType`|`body`|`string`|否|设备系列|
|`++deviceModel`|`body`|`string`|否|设备型号|
|`++deviceKeyVersion`|`body`|`string`|否|设备驱动版本号|
|`++devSerialNum`|`body`|`string`|否|设备的序列号|
|`++deviceIndexCode`|`body`|`string`|否|主动设备编号|
|`++manufacturer`|`body`|`string`|否|厂商|
|`++zoneId`|`body`|`string`|否|时区ID|
|`++netZoneId`|`body`|`string`|否|所属网域|
|`++deviceCode`|`body`|`string`|否|设备机号|
|`++cardCapacity`|`body`|`number`|否|设备卡容量|
|`++faceCapacity`|`body`|`number`|否|人脸容量|
|`++description`|`body`|`string`|否|描述|
|`++regionIndexCode`|`body`|`string`|否|所属区域|
|`++regionPath`|`body`|`string`|否|所属区域目录,以@符号分割，包含本节点|
|`++treatyType`|`body`|`string`|否|接入协议，详见附录A.6 编码设备接入协议|
|`++installLocation`|`body`|`string`|否|安装位置|
|`++createTime`|`body`|`string`|否|创建时间|
|`++updateTime`|`body`|`string`|否|更新时间|
|`++disOrder`|`body`|`number`|否|数据在界面上的显示顺序|
|`++regionName`|`body`|`string`|否|所属区域目录，由唯一标示组成，最大10级，格式： @根节点@子区域1@子区域2@|
|`++regionPathName`|`body`|`string`|否|区域目录名称，"/"分隔|
