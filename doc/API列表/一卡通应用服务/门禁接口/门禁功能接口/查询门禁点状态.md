# 查询门禁点状态

> 接口说明
> 该接口支持门常开、门常闭、门开和门闭四种操作引起的门状态获取。门常开操作，门会一直处于开状态，不会自动关闭，执行门闭操作，门才会关上；门常闭操作，门会一直处于关毕状态，普通卡刷卡门不会被打开，执行门开操作，门会打开；门开操作，执行门打开动作，超过门打开时间，门会自动关上；门闭操作，执行关门动作，会立即把门关上。
> 调用该接口，首先要通过获取门禁点资源列表的接口，获取到门禁点唯一编号，然后根据门禁点唯一编号进行门禁点状态状态查询。需要注意的是门通道必须接上门磁才能正常发送门状态变化通知，如果未接门磁，平台无法通过门状态变更通知来更新门状态。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acs/v1/door/states`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acs/v1/door/states
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`doorIndexCodes`|`body`|`string[]`|否|门禁点唯一标识，最大支持200个门禁点。查询门禁点列表v2接口获取返回参数doorIndexCode|

## 请求参数示例
```json
{
  "doorIndexCodes": [
    "1f276203e5234bdca08f7d99e1097bba"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其它参考附录E.2.1 门禁管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+authDoorList`|`body`|`object[]`|否|有权限的门禁点状态集合|
|`++doorIndexCode`|`body`|`string`|否|门禁点indexCode|
|`++doorState`|`body`|`number`|否|门状态，0 初始状态，1 开门状态，2关门状态，3离线状态|
|`+noAuthDoorIndexCodeList`|`body`|`string[]`|否|没有权限的门禁点集合|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "authDoorList": [
      {
        "doorIndexCode": "e8e3ef5c149243abb4341124ab38fcfc",
        "doorState": 0
      }
    ],
    "noAuthDoorIndexCodeList": [
      "e8e3ef5c149243abb4341124ab38fcfc"
    ]
  }
}
```
