# 查询门禁点事件v2

> 接口说明
> 功能描述：该接口可以查询发生在门禁点上的人员出入事件，支持多个维度来查询，支持按时间、人员、门禁点、事件类型四个维度来查询；其中按事件类型来查询的方式，如果查询不到事件，存在两种情况，一种是该类型的事件没有发生过，所以查询不到，还有一种情况，该类型的事件发生过，但是由于门禁管理组件对该事件类型订阅配置处于关闭状态，所以不会存储该类型的事件，导致查询不到，对于这种情况，需要到门禁管理组件中，将该事件类型的订阅配置打开。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/acs/v2/door/events`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acs/v2/door/events
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`pageNo`|`body`|`integer`|是|当前页码（pageNo>0）|
|`pageSize`|`body`|`integer`|是|每页展示数目（0<pageSize<=1000）|
|`doorIndexCodes`|`body`|`string[]`|否|门禁点唯一标识数组，最大支持10个门禁点，查询门禁点列表v2接口获取返回报文中的doorIndexCode字段|
|`doorName`|`body`|`string`|否|门禁点名称，支持模糊查询，从查询门禁点列表v2接口获取返回报文中的name字段|
|`readerDevIndexCodes`|`body`|`string[]`|否|读卡器唯一标识数组，最大支持50个读卡器，查询门禁读卡器列表接口获取返回报文中的indexCode字段|
|`startTime`|`body`|`string`|否|开始时间（事件开始时间，采用ISO8601时间格式，与endTime配对使用，不能单独使用，时间范围最大不能超过3个月，与receiveStartTime二选一必填），参考附录B ISO8601时间格式说明|
|`endTime`|`body`|`string`|否|结束时间（事件结束时间，采用ISO8601时间格式，最大长度32个字符，与startTime配对使用，不能单独使用，时间范围最大不能超过3个月，与receiveEndTime二选一必填），参考附录B ISO8601时间格式说明|
|`receiveStartTime`|`body`|`string`|否|入库开始时间，采用ISO8601时间格式，与receiveEndTime配对使用，不能单独使用，时间范围最大不能超过3个月，与startTime二选一必填，参考附录B ISO8601时间格式说明|
|`receiveEndTime`|`body`|`string`|否|入库结束时间，采用ISO8601时间格式，最大长度32个字符，与receiveStartTime配对使用，不能单独使用，时间范围最大不能超过3个月，与endTime二选一必填，参考附录B ISO8601时间格式说明|
|`doorRegionIndexCodes`|`body`|`string[]`|否|门禁点所在区域列表，最大支持500个区域|
|`eventTypes`|`body`|`integer[]`|否|事件类型，参考[附录D2.1 门禁事件](/docs/docId?productId=5c67f1e2f05948198c909700&version=/29c78ef52ca842c7933bd2b8e051e9d0&tagPath=附录-附录D%20事件列表-附录D2 一卡通事件-附录D2.1 门禁事件)|
|`personName`|`body`|`string`|否|人员姓名(支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`personIds`|`body`|`string[]`|否|人员数组（最大支持100个人员）|
|`sort`|`body`|`string`|否|排序字段（支持personName、doorName、eventTime填写排序的字段名称）|
|`order`|`body`|`string`|否|升/降序（指定排序字段是使用升序（asc）还是降序（desc）|

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0-成功，其它参考附录E.2.1 门禁管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|数据总数|
|`+totalPage`|`body`|`number`|否|总页数|
|`+pageNo`|`body`|`number`|否|页码|
|`+pageSize`|`body`|`number`|否|页大小|
|`+list`|`body`|`object[]`|否|返回数据集合|
|`++eventId`|`body`|`string`|否|事件ID，唯一标识这个事件|
|`++eventName`|`body`|`string`|否|事件名称|
|`++eventTime`|`body`|`string`|否|事件产生时间(事件产生的时间，采用ISO8601时间格式)|
|`++personId`|`body`|`string`|否|人员唯一编码|
|`++cardNo`|`body`|`string`|否|卡号|
|`++personName`|`body`|`string`|否|人员姓名|

|`++orgName`|`body`|`string`|否|人员所属组织名称|
|`++doorName`|`body`|`string`|否|门禁点名称|
|`++doorIndexCode`|`body`|`string`|否|门禁点编码|
|`++doorRegionIndexCode`|`body`|`string`|否|门禁点所在区域编码|
|`++picUri`|`body`|`string`|否|抓拍图片地址(抓拍图片uri，它是一个相对地址，可以通过“获取门禁事件抓拍的图片”的接口，获取到图片的数据)|
|`++svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识(与picUri配对输出的字段信息，用于“获取门禁事件抓拍的图片”接口的输入参数)|
|`++eventType`|`body`|`number`|否|事件类型|
|`++inAndOutType`|`body`|`number`|否|进出类型(1：进|
|`++readerDevIndexCode`|`body`|`string`|否|读卡器唯一标识|
|`++readerDevName`|`body`|`string`|否|读卡器名称|
|`++devIndexCode`|`body`|`string`|否|控制器设备唯一标识|
|`++devName`|`body`|`string`|否|控制器设备名称|
|`++identityCardUri`|`body`|`string`|否|身份证图片地址(身份证图片uri，可以通过“获取门禁事件抓拍的图片”的接口，获取到图片的数据)|
|`++receiveTime`|`body`|`string`|否|事件入库时间，采用ISO8601时间格式|
|`++jobNo`|`body`|`string`|否|工号|
|`++studentId`|`body`|`string`|否|学号|
|`++certNo`|`body`|`string`|否|证件号码|

补充说明:
- 0：出
- -1:未知
- 要求：进门读卡器拨码设置为1，出门读卡器拨码设置为2
- )
