# 访客权限重新下发

> 接口说明
> 该接口用于对访客的权限进行重新下发。当访客登记时或者访客签离时，由于一些不确定的原因导致权限下载（或者取消）失败，可以通过该接口实现对访客的这些失败的权限进行再试下发。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/visitor/v1/auth/reapplication`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v1/auth/reapplication
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orderId`|`body`|`string`|是|访客记录id，登记时获取，或从查询访客来访记录v2接口获取返回报文中的orderId字段|

## 请求参数示例
```json
{
  "orderId": "123"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`string`|否|返回数据|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": "null"
}
```
