# 预约免登记

> 接口说明
> 本接口提供在完成访客预约操作后，为访客下发默认权限组的权限和入参“权限下发指定的资源点集合”指定的资源的权限的能力，本接口支持上传人脸照片，提供人脸权限下载的能力。预约免登记操作后的记录状态为正常或者迟到，当访客进行园区后，访客管理应用服务组件收到访客相关的刷卡、刷脸事件、车辆出入事件后将访客记录状态改为已到达或者超期未签离，完成访客自动登记流程操作。如果仅配置了默认权限组，则只按照默认权限组进行权限下发；如果仅指定权限下发的资源点集合，则只按照资源点集合进行权限下发；如果两者都为空，则不进行权限下发操作。
> 预约免登记接口创建的预约记录支持预约记录的修改、取消和登记操作。但修改操作只能对访客的基本信息进行修改，不能对访客的权限信息进行修改。修改操作使用“修改访客预约”接口，取消操作使用“取消访客预约接口”；如果访客到访，但还未进行任何的刷卡、刷脸事件、车辆出入操作，则可以对该预约免登记记录进行登记接口，如果访客产生了刷卡、刷脸事件、车辆出入操作，则访客管理应用服务会自动进行登记操作，接口应用方将不能进行登记操作。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/visitor/v1/appointment/registration`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v1/appointment/registration
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`receptionistId`|`body`|`string`|是|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`visitStartTime`|`body`|`string`|是|预计来访时间，时间参数需满足ISO8601格式： yyyy-MM-ddTHH:mm:ss+当前时区， 例如北京时间： 2018-07-26T15:00:00 + 08:00|
|`visitEndTime`|`body`|`string`|是|预计离开时间，时间参数需满足ISO8601格式： yyyy-MM-ddTHH:mm:ss+当前时区， 例如北京时间： 2018-07-26T15:00:00 + 08:00 预计离开时间必须晚于当前时间和预计来访时间|
|`visitPurpose`|`body`|`string`|否|来访事由，长度为0～128个字符|
|`personNum`|`body`|`integer`|否|来访人数，指同一批来访者的总人数，即主访客+随行人数|
|`parkId`|`body`|`string`|否|园区id,不填使用默认园区|
|`visitorInfo`|`body`|`object`|是|访客信息|
|`+visitorName`|`Body`|`string`|是|访客姓名，支持1-32位中英文字符，不能包含 ‘ / \ : * ? “ < >|
|`+phoneNo`|`Body`|`string`|是|联系电话建议填写手机号码，仅支持1-20位纯数字|
|`+plateNo`|`Body`|`string`|否|车牌号，1-16位，不能包含 ‘ / \ : * ? “ < >|
|`+certificateType`|`Body`|`number`|否|证件类型，详见附录A.11 证件类型|
|`+certificateNo`|`Body`|`string`|否|证件号码，1~20个数字、字母组成；证件号码非空时，证件类型必填|
|`+certAddr`|`Body`|`string`|否|证件地址，支持1-128位字母、汉字|
|`+certIssuer`|`Body`|`string`|否|发证机关，支持1-32位字母、汉字|
|`+birthplace`|`Body`|`string`|否|籍贯，支持1-32位字母、汉字|
|`+visitorWorkUnit`|`Body`|`string`|否|来访单位，支持1-32位字符，不包含’/😗?”<>|
|`+cardNo`|`Body`|`string`|否|卡号,8-20位数字、大写字母|
|`+visitorPhoto`|`Body`|`string`|否|头像base64编码的字节流，图片最大200K，仅支持jpg格式。 访客头像需要秘钥加密传输，由于访客头像需要下发到设备，因此该接口会对访客头像进行质量检测，只有人脸评分大于等于75时，接口才会返回成功|
|`+visitorAddress`|`Body`|`string`|否|访客住址，支持1-128位中英文数字|
|`+visitorTypeId`|`Body`|`string`|否|访客类型id|
|`+plateColor`|`Body`|`number`|否|车牌颜色 0-蓝色 1-黄色 2-白色 3-黑色 4-绿色 5-民航黑色 255-其他|
|`visitorPermissionSet`|`body`|`object`|否|访客权限|
|`+privilegeGroupIds`|`Body`|`string[]`|否|访客权限组的id集合，限制最多传入5个权限组，查询访客权限组 接口获取返回报文中的privilegeGroupId字段|
|`+defaultPrivilegeGroupFlag`|`Body`|`string`|否|是否使用默认权限组,1：使用；非1：不使用|
|`+designatedResources`|`Body`|`object[]`|否|资源点集合，数量不能超过1000|
|`++paramKey`|`Body`|`string`|否|参数类型(1:门禁点2:梯控楼层3:门口机4:围墙机5:停车场7:车道)|
|`++paramValues`|`Body`|`string[]`|否|参数集合(资源编码)|

补充说明:
- 默认为1，不能小于1大于999

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+appointRecordId`|`body`|`string`|否|预约记录ID|
|`+visitorName`|`body`|`string`|否|访客姓名|
|`+receptionistId`|`body`|`string`|否|被访人唯一标识|
|`+receptionistName`|`body`|`string`|否|被访人姓名|
|`+verificationCode`|`body`|`string`|否|访客验证码,为访客组件提供，用于访客在访客机登记的凭证|
|`+orderId`|`body`|`string`|否|访客记录唯一标识，用于访客签离、已预约登记等操作|
|`+QRCode`|`body`|`string`|否|动态二维码内容|
|`+picUri`|`body`|`string`|否|访客头像，返回图片的相对Uri|
|`+svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识|
|`+visitorInfoId`|`body`|`string`|否|访客信息唯一标识|
