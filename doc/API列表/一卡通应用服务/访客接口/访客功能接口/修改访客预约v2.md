# 修改访客预约v2

> 接口说明
> 根据“访客预约”接口返回的预约记录ID作为唯一标识，来修改访客的预约信息。可修改的信息包括被访人、预计来访时间、预计离开时间、来访事由，以及访客的详细信息。
> 修改访客预约信息的处理原则是先删除原有的访客信息，然后添加更新的访客信息，所以会重新生成新的访客ID，但是预约记录ID保持不变。
> 注：当某个被访人有多个访客，但只需要修改其中一个访客信息时，基于“修改访客预约”接口的删除再添加的原则，需要对这个被访人的所有访客进行重新预约。修改的访客预约记录必须有效，即预约记录ID真实存在，且对应的预约状态是正常或者迟到的任何一种，其中正常状态，表示访客还没登记，且预计来访时间晚于当前时间，迟到状态，表示访客还没登记，预计来访时间早于当前时间，且预计离开时间晚于当前时间。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/visitor/v2/appointment/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v2/appointment/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`appointRecordId`|`body`|`string`|是|预约记录ID，可从查询访客预约记录v2接口获取返回报文中的appointRecordId字段|
|`receptionistId`|`body`|`string`|是|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`visitStartTime`|`body`|`string`|是|时间参数需满足ISO8601格式：|
|`visitEndTime`|`body`|`string`|是|时间参数需满足ISO8601格式：yyyy-MM-ddTHH:mm:ss+当前时区，|
|`visitPurpose`|`body`|`string`|否|来访事由，长度为0～32个字符|
|`visitorInfoList`|`body`|`object[]`|是|访客信息列表|
|`+visitorName`|`Body`|`string`|是|访客姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`+phoneNo`|`Body`|`string`|是|联系电话建议填写手机号码，仅支持纯数字|
|`+plateNo`|`Body`|`string`|否|车牌号,不能包含 ’ / \ : * ? " < >|
|`+certificateType`|`Body`|`number`|否|证件类型，详见附录A.11 证件类型|
|`+certificateNo`|`Body`|`string`|否|证件号码，1~20个数字、字母组成；|
|`+certAddr`|`Body`|`string`|否|证件地址，支持1-128位字母、汉字|
|`+certIssuer`|`Body`|`string`|否|发证机关，支持1-32位字母、汉字|
|`+birthplace`|`Body`|`string`|否|籍贯，支持1-32位字母、汉字|
|`+visitorWorkUnit`|`Body`|`string`|否|来访单位，支持1-32位字符，不包含’/😗?”<>|
|`+visitorPhoto`|`Body`|`string`|否|头像base64编码的字节流，图片最大200K，仅支持jpg格式。 访客头像需要秘钥加密传输，由于访客头像需要下发到设备，因此该接口会对访客头像进行质量检测，只有人脸评分大于等于75时，接口才会返回成功|
|`+visitorAddress`|`Body`|`string`|否|访客住址，支持1-128位中英文数字|
|`+plateColor`|`Body`|`number`|否|车牌颜色 0-蓝色 1-黄色 2-白色 3-黑色 4-绿色 5-民航黑色 255-其他|
|`downloadParkAuthTag`|`body`|`integer`|否|只针对通过api接口添加的预约免登记的记录才有效；下发停车场权限的标志: 0:不下发 1:下发 不填，按照通过预约免登记添加接口设置参数为准|
|`receptionistOrgCode`|`body`|`string`|否|被访人所属组织code|
|`visitorPermissionSet`|`body`|`object`|否|权限集合封装对象|
|`+defaultPrivilegeGroupFlag`|`Body`|`string`|否|是否使用默认权限组(1：使用；非1：不使用)|
|`+privilegeGroupIds`|`Body`|`string[]`|否|权限组id集合,数量不能超过20|
|`+designatedResources`|`Body`|`object[]`|否|资源点集合，数量不能超过1000|
|`++paramKey`|`Body`|`string`|否|参数类型(1:门禁点2:梯控楼层3:门口机4:围墙机5:停车场7:车道)|
|`++paramValues`|`Body`|`string[]`|否|参数集合(资源编码)|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，
- 例如北京时间：
- 2018-07-26T15:00:00+08:00
- 例如北京时间：
- 2018-07-26T15:00:00+08:00预计离开时间必须晚于当前时间和预计来访时间
- 证件号码非空时，证件类型必填

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述，与返回码对应|
|`data`|`body`|`object`|否|返回数据|
|`+appointRecordId`|`body`|`string`|否|预约单唯一ID，可作为修改预约的依据|
|`+appointmentInfoList`|`body`|`object[]`|否|预约详细信息|
|`++visitorName`|`body`|`string`|否|支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`++receptionistId`|`body`|`string`|否|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`++receptionistName`|`body`|`string`|否|支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`++verificationCode`|`body`|`string`|否|访客验证码为访客组件提供，用于访客在访客机登记的凭证|
|`++QRCode`|`body`|`string`|否|动态二维码，如需要使用，可将访客二维码内容转成二维码图片即可|
|`++orderId`|`body`|`string`|否|访客记录唯一标识，用于访客签离、已预约登记等操作|
|`++picUri`|`body`|`string`|否|访客头像,返回图片的相对Uri|
|`++svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识|
|`++visitorInfoId`|`body`|`string`|否|访客信息唯一标识|
