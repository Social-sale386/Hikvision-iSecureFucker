# 已预约登记

> 接口说明
> 该接口主要是应用于对已预约的访客进行登记和授权。当访客已经预约时，访客管理应用服务组件可以通过访客到访时提供的二维码、验证码等信息调用“查询访客预约记录”接口获取访客的预约信息，然后根据预约Id等信息调用本接口实现访客的登记与权限授予操作。该接口的权限授予功能依赖于“查询访客权限组”接口，用户需要先在访客管理应用服务中配置好访客权限组。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/visitor/v1/order/register`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v1/order/register
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orderId`|`body`|`string`|是|预约记录ID，可从查询访客预约记录v2接口获取返回报文中的orderId字段|
|`visitEndTime`|`body`|`string`|是|预计离开时间，时间参数需满足ISO8601格式：yyyy-MM-ddTHH:mm:ss+当前时区，例如北京时间：|
|`cardNo`|`body`|`string`|否|卡号|
|`visitorPermissionSet`|`body`|`object`|否|访客权限|
|`+privilegeGroupIds`|`Body`|`string[]`|否|权限组ids，查询访客权限组 接口获取返回报文中的privilegeGroupId字段,权限组id集合,数量不能超过20|
|`+defaultPrivilegeGroupFlag`|`Body`|`string`|否|是否使用默认权限组，1：使用；非1：不使用|
|`+designatedResources`|`Body`|`object[]`|否|资源点集合，数量不能超过1000|
|`++paramKey`|`Body`|`string`|否|参数类型(1:门禁点2:梯控楼层3:门口机4:围墙机5:停车场7:车道)|
|`++paramValues`|`Body`|`string[]`|否|参数集合(资源编码)|
|`isUseOrderAuth`|`body`|`string`|否|是否使用预约时的权限：1：是（预约时权限和登记时的权限累加）0：否（使用登记时的权限集合）;默认为0|

补充说明:
- 2018-07-26T15:00:00+08:00
- 预计离开时间必须晚于当前时间和预计来访时间

## 请求参数示例
```json
{
  "orderId": "d5d83be0-76d3-11e9-aed0-73c3f56257ef",
  "visitEndTime": "2019-07-26T19:00:00+08:00",
  "cardNo": "55446774",
  "visitorPermissionSet": {
    "privilegeGroupIds": [
      "23436"
    ],
    "defaultPrivilegeGroupFlag": "1",
    "designatedResources": [
      {
        "paramKey": "1",
        "paramValues": [
          "aaaaaaaaaaa"
        ]
      }
    ]
  },
  "isUseOrderAuth": "0"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|访客登记信息|
|`+QRCode`|`body`|`string`|否|动态二维码内容|
|`+orderId`|`body`|`string`|否|访客记录唯一标识，用于访客签离、已预约登记等操作|
|`+svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识|
|`+picUri`|`body`|`string`|否|访客头像，返回图片的相对Uri；需要调用“获取访客记录的图片”接口，来获取图片的数据|
