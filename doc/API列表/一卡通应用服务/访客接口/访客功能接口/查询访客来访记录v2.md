# 查询访客来访记录v2

> 接口说明
> 访客来访记录指访客登记后的访客信息，访客预约记录在访客机上完成登记操作，就会变成访客来访记录。
> 查询访客来访记录接口支持分页查询，可以根据查询条件：访客姓名、被访人、来访时间、离开时间、访客联系电话、访客记录ID、访客状态进行组合查询，接口需要传入用户登录名，用于权限过滤。其中访客姓名、被访人姓名、证件号码和联系电话支持模糊查询。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/visitor/v2/visiting/records`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v2/visiting/records
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`receptionistId`|`body`|`string`|否|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`receptionistName`|`body`|`string`|否|被访人姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`visitorName`|`body`|`string`|否|访客姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`phoneNo`|`body`|`string`|否|访客联系电话建议填写手机号码|
|`visitorStatusList`|`body`|`array`|否|访客状态集合:5：超期自动签离,6：已签离,7：超期未签离,8：已到达|
|`visitStartTimeBegin`|`body`|`string`|否|来访时间查询时间段条件的开始时间，时间参数需满足ISO8601格式：|
|`visitStartTimeEnd`|`body`|`string`|否|来访时间查询时间段条件的结束时间，时间参数需满足ISO8601格式：yyyy-MM-ddTHH:mm:ss+当前时区，|
|`visitEndTimeBegin`|`body`|`string`|否|离开时间查询时间段条件的开始时间，时间参数需满足ISO8601格式：|
|`visitEndTimeEnd`|`body`|`string`|否|离开时间查询时间段条件的结束时间，时间参数需满足ISO8601格式：yyyy-MM-ddTHH:mm:ss+当前时区，|
|`pageNo`|`body`|`integer`|是|pageNo要求大于0不超过2147483647|
|`pageSize`|`body`|`integer`|是|pageSize要求大于0且不超过1000|
|`cardNo`|`body`|`string`|否|卡号，8-20位数字、大写字母|
|`orderId`|`body`|`string`|否|访客记录id，登记时获取|
|`QRCode`|`body`|`string`|否|二维码内容，支持动态二维码或者静态二维码|
|`identityNum`|`body`|`string`|否|证件号码|
|`parkId`|`body`|`string`|否|园区编号|
|`plateNo`|`body`|`string`|否|访客车牌号|
|`receptionistCode`|`body`|`string`|否|被访人所属组织编号|
|`visitUpdateTimeBegin`|`body`|`string`|否|更新时间查询时间段条件的开始时间|
|`visitUpdateTimeEnd`|`body`|`string`|否|更新来访时间查询时间段条件的结束时间|
|`visitorInfoId`|`body`|`string`|否|访客信息唯一标识|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，
- 例如北京时间：
- 2018-07-26T15:00:00+08:00
- 例如北京时间：
- 2018-07-26T15:00:00+08:00预来访时间的结束时间必须晚于来访时间的开始时间
- yyyy-MM-ddTHH:mm:ss+当前时区，
- 例如北京时间：
- 2018-07-26T15:00:00+08:00
- 例如北京时间：
- 2018-07-26T15:00:00+08:00离开时间必须晚于离开时间的开始时间
