# 生成访客动态二维码内容

> 接口说明
> 该接口用于查询访客的动态二维码内容，应用方可以获取该二维码内容，然后生成二维码图片，最后使用该二维码在门禁等设备上签权或者使用该二维码查询访客信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/visitor/v1/auth/qcode`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v1/auth/qcode
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orderId`|`body`|`string`|是|访客记录id，登记时获取，或从查询访客来访记录v2接口获取返回报文中的orderId字段|
|`duration`|`body`|`integer`|是|有效时长:有效期为当前时间+有效时长，参数单位由unit参数决定，如:有效时长参数为1，unit参数为1，则有效时长为当前时间+1小时，|
|`unit`|`body`|`integer`|是|时间单位，1：小时；2：分钟|
|`frequency`|`body`|`integer`|否|有效次数，不填默认1次（单个设备上使用的有效次数）|

补充说明:
- 当有效期超过访客的预计离开时间时，系统默认有效期为访客的预计离开时间。

## 请求参数示例
```json
{
  "orderId": "0897408189748124",
  "duration": 1,
  "unit": 1,
  "frequency": 1
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`string`|否|动态二维码内容|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "sucess",
  "data": "VjAwMSvtAyPTotthmgasIJD4Uafp9v8WpaQmhK1Fw4UaLIMk+wzkj7fZ1fUXJArpm8t9qew="
}
```
