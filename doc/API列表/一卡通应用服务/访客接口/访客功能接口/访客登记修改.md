# 访客登记修改

> 接口说明
> 本接口提供修改访客登记信息能力。本接口支持访客信息的自定义字段，按照属性定义key:value上传到访客信息列表：visitorInfoList中即可， 通过获取访客基础信息字段接口获取平台已支持访客自定义属性的说明。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/visitor/v1/order/update`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v1/order/update
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`orderId`|`body`|`string`|是|访客记录id，登记时获取，或从查询访客来访记录v2接口获取返回报文中的orderId字段|
|`endTimeISO`|`body`|`string`|是|预计离开时间,ISO格式|
|`visitPurpose`|`body`|`string`|否|来访事由,长度为1～128个字符|
|`receptionistId`|`body`|`string`|否|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`visitorPermissionSet`|`body`|`object`|否|权限集合封装对象|
|`+defaultPrivilegeGroupFlag`|`Body`|`string`|否|是否使用默认权限组(1：使用；非1：不使用)|
|`+privilegeGroupIds`|`Body`|`string[]`|否|访客权限组的id，查询访客权限组 接口获取返回报文中的privilegeGroupId字段|
|`+designatedResources`|`Body`|`object[]`|否|资源点集合，数量不能超过1000|
|`++paramKey`|`Body`|`string`|否|参数类型(1:门禁点2:梯控楼层3:门口机4:围墙机5:停车场7:车道)|
|`++paramValues`|`Body`|`string[]`|否|参数集合(资源编码)|
|`visitorInfo`|`body`|`object`|是|访客详细信息|
|`+birthplace`|`Body`|`string`|否|籍贯，支持1-32位字母、汉字|
|`+certAddr`|`Body`|`string`|否|证件地址，支持1-128位字母、汉字|
|`+certIssuer`|`Body`|`string`|否|发证机关，支持1-32位字母、汉字|
|`+visitorAddress`|`Body`|`string`|否|访客住址，支持1-128位中英文数字|
|`+visitorName`|`Body`|`string`|是|访客姓名，支持中英文字符，不能包含 ‘ / \ : * ? “ < >|
|`+visitorPhoto`|`Body`|`string`|否|头像base64编码的字节流，图片最大200K，仅支持jpg格式。 由于访客头像需要下发到设备，因此该接口会对访客头像进行质量检测，只有人脸评分大于等于75时，接口才会返回成功，通过人脸评分接口获取人脸图片的评分|
|`+visitorWorkUnit`|`Body`|`string`|否|来访单位，支持1-32位字符，不包含’/😗?”<>|
|`+phoneNo`|`Body`|`string`|是|联系电话建议填写手机号码，仅支持1-20位纯数字|
|`+plateNo`|`Body`|`string`|否|车牌号，1-16位，不能包含 ‘ / \ : * ? “ < >|
|`+certificateType`|`Body`|`number`|否|证件类型，详见附录A.11 证件类型|
|`+certificateNo`|`Body`|`string`|否|证件号码，1~20个数字、字母组成；证件号码非空时，证件类型必填|
|`+plateColor`|`Body`|`number`|否|车牌颜色 0-蓝色 1-黄色 2-白色 3-黑色 4-绿色 5-民航黑色 255-其他|
|`registerPlace`|`body`|`string`|否|访客登记地点(长度不超过1024)|
|`receptionistOrgCode`|`body`|`string`|否|被访人所属组织code|
|`cardNo`|`body`|`string`|否|卡号,8-20位数字|

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|访客登记信息|
|`+QRCode`|`body`|`string`|否|动态二维码内容|
|`+orderId`|`body`|`string`|否|访客记录唯一标识，用于访客签离、已预约登记等操作|
|`+svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识|
|`+picUri`|`body`|`string`|否|访客头像，返回图片的相对Uri；需要调用“获取访客记录的图片”接口，来获取图片的数据|
