# 访客预约v2

> 接口说明
> 本接口用于为访客提供预约的能力，预约成功后，每个访客都有独立的验证码和二维码本接口支持访客信息的自定义字段，按照属性定义key:value上传到访客信息列表：visitorInfoList中即可， 通过【获取访客基础信息字段接口】获取平台已支持访客自定义属性的说明。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/visitor/v2/appointment`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/visitor/v2/appointment
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`receptionistId`|`body`|`string`|是|被访人唯一标识,查询人员列表v2接口获取返回报文中的personId字段|
|`visitorPermissionSet`|`body`|`object`|否|权限集合封装对象|
|`+defaultPrivilegeGroupFlag`|`Body`|`string`|否|是否使用默认权限组(1：使用；非1：不使用)|
|`+privilegeGroupIds`|`Body`|`string[]`|否|权限组ids，限制最多传入5个权限组，查询访客权限组 接口获取返回报文中的privilegeGroupId字段|
|`+designatedResources`|`Body`|`object[]`|否|资源点集合，数量不能超过1000|
|`++paramKey`|`Body`|`string`|否|参数类型(1:门禁点2:梯控楼层3:门口机4:围墙机5:停车场7:车道)|
|`++paramValues`|`Body`|`string[]`|否|参数集合(资源编码)|
|`visitStartTime`|`body`|`string`|是|预计来访时间，时间参数需满足ISO8601格式：|
|`visitEndTime`|`body`|`string`|是|预计离开时间，时间参数需满足ISO8601格式：|
|`visitPurpose`|`body`|`string`|否|来访事由，长度为0～32个字符|
|`visitorInfoList`|`body`|`object[]`|是|访客信息列表，最大支持128位访客，其中最多支持4位访客带人脸信息，如果存在人脸上传失败，则整个预约失败。|
|`+visitorName`|`Body`|`string`|是|访客姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`+phoneNo`|`Body`|`string`|是|联系电话，建议填写手机号码，仅支持纯数字|
|`+plateNo`|`Body`|`string`|否|车牌号，不能包含 ’ / \ : * ? " < >|
|`+certificateType`|`Body`|`number`|否|证件类型，详见附录A.11 证件类型|
|`+certificateNo`|`Body`|`string`|否|证件号码，1~20个数字、字母组成；|
|`+certAddr`|`Body`|`string`|否|证件地，支持1-128位字母、汉字|
|`+certIssuer`|`Body`|`string`|否|发证机关，支持1-32位字母、汉字|
|`+birthplace`|`Body`|`string`|否|籍贯，支持1-32位字母、汉字|
|`+visitorPhoto`|`Body`|`string`|否|头像base64编码的字节流，图片最大200K，仅支持jpg格式。|
|`+visitorWorkUnit`|`Body`|`string`|否|来访单位，支持1-32位字符，不包含’/😗?”<>|
|`+visitorAddress`|`Body`|`string`|否|访客住址，支持1-128位中英文数字|
|`+extensions`|`Body`|`object`|否|扩展字段集合,json格式：以key:value的json字符串形式填入（其中key与平台自定义字段相对应,可通过"获取访客字段的配置信息"接口获取,返回数据中的"name"字段作为key;扩展字段不支持图片类型|
|`+plateColor`|`Body`|`number`|否|车牌颜色 0-蓝色 1-黄色 2-白色 3-黑色 4-绿色 5-民航黑色 255-其他|
|`parkId`|`body`|`string`|否|园区id,不填使用默认园区|
|`receptionistOrgCode`|`body`|`string`|否|被访人所属组织code|
|`visitorTypeId`|`body`|`string`|否|访客类型id|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，
- 例如北京时间：
- 2018-07-26T15:00:00 + 08:00
- yyyy-MM-ddTHH:mm:ss+当前时区，
- 例如北京时间：
- 2018-07-26T15:00:00 + 08:00
- 预计离开时间必须晚于当前时间和预计来访时间
- 证件号码非空时，证件类型必填；
- 由于访客头像需要下发到设备，因此该接口会对访客头像进行质量检测，只有人脸评分大于等于75时，接口才会返回成功

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.2 访客管理错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|访客预约结果信息|
|`+appointRecordId`|`body`|`string`|否|预约记录ID，可作为修改预约的依据|
|`+appointmentInfoList`|`body`|`object[]`|否|预约详细信息|
|`++visitorName`|`body`|`string`|否|访客姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`++receptionistId`|`body`|`string`|否|被访人唯一标识|
|`++receptionistName`|`body`|`string`|否|被访人姓名，支持中英文字符，不能包含 ’ / \ : * ? " < >|
|`++verificationCode`|`body`|`string`|否|访客验证码，可用于访客在访客机登记的凭证， 或后期作为查询条件使用|
|`++QRCode`|`body`|`string`|否|动态二维码内容，|
|`++orderId`|`body`|`string`|否|访客记录唯一标识，用于访客签离、已预约登记等操作|
|`++picUri`|`body`|`string`|否|访客头像,返回图片的相对Uri|
|`++svrIndexCode`|`body`|`string`|否|图片存储服务的唯一标识|
|`++visitorInfoId`|`body`|`string`|否|访客信息唯一标识|

补充说明:
- 如需要使用，将访客二维码内容转成二维码图片即可
