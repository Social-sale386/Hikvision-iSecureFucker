# 查询梯控历史事件信息

> 接口说明
> 该接口可以查询发生在电梯上的人员出入事件和设备事件，支持多个维度来查询，支持按时间、人员、设备、卡号、事件类型五个维度来查询；其中按事件类型来查询的方式，如果查询不到事件，存在两种情况，一种是该类型的事件没有发生过，所以查询不到，还有一种情况，该类型的事件发生过，但是由于梯控管理组件对该事件类型订阅配置处于关闭状态，所以不会存储该类型的事件，导致查询不到，对于这种情况，需要到系统配置->梯控管理组件中，将该事件类型的订阅配置打开。
> 事件发生时间:eventTime指事件在设备上产生的时间,平台接收事件时间:receiveTime指平台接收到事件的时间,两者可能有几秒的差别,如果用该接口获取事件来做考勤,应该通过receiveTime作为条件获取事件信息,保存在本地,然后通过receiveTime做考勤,这样可以保证数据不会丢失。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/ecs/v1/access_events/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/ecs/v1/access_events/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`startTime`|`body`|`string`|是|事件开始时间，采用ISO8601时间格式，与endTime配对使用，不能单独使用，时间范围最大不能超过3个月(90天)|
|`endTime`|`body`|`string`|是|事件结束时间，采用ISO8601时间格式，最大长度32个字符，与startTime配对使用，不能单独使用，时间范围最大不能超过3个月(90天)|
|`deviceCodes`|`body`|`string`|否|设备id组(最大支持500个设备集搜索,多个用”,”隔开)|
|`personIds`|`body`|`string`|否|人员id组(人员Id通过查询人员信息接口获取，最大支持1000个人员搜索,多个用”,”隔开)|
|`cardNos`|`body`|`string`|否|卡片组(最大支持1000张卡片搜索,多个用”,”隔开)|
|`eventTypes`|`body`|`string`|否|事件类型，10进制，多个事件类型用”,”隔开，参考附录D.1 事件类型|
|`sort`|`body`|`string`|否|排序字段，支持personName、deviceName、eventTime，填写排序的字段名称|
|`order`|`body`|`string`|否|升/降序，指定排序字段是使用升序（asc）还是降序（desc）|
|`pageNo`|`body`|`integer`|是|当前页码，当前页码需大于0（pageNo>0）|
|`pageSize`|`body`|`integer`|是|每页记录总数，每页记录展示的数目应大于0，小于等于1000（0<pageSize<=1000）|

## 请求参数示例

```json
{
  "startTime": "2019-02-27T12:00:00.000+08:00",
  "endTime": "2019-03-01T12:00:00.000+08:00",
  "deviceCodes": "620ebebe-c548-4144-8896-aa12c6c0a8cd,b0454230-0f40-4b49-9da1-5e6e230f094",
  "personIds": "a70f4c402044b89335529e0,1afc74a7437ecf47ae78b05",
  "cardNos": "2666425645,1873925518",
  "eventTypes": "723202,721922",
  "sort": "personName",
  "order": "asc",
  "pageNo": 1,
  "pageSize": 100
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+pageSize`|`body`|`number`|否|页大小|
|`+total`|`body`|`number`|否|总数|
|`+totalPage`|`body`|`number`|否|总页数|
|`+page`|`body`|`number`|否|页码|
|`+rows`|`body`|`object[]`|否|事件信息|
|`++eventId`|`body`|`string`|否|事件ID，唯一标识这个事件|
|`++eventType`|`body`|`number`|否|事件类型，10进制，参考附录D.1 事件类型|
|`++eventName`|`body`|`string`|否|事件名称|
|`++eventTime`|`body`|`string`|否|事件产生的时间，采用ISO8601时间格式|
|`++personId`|`body`|`string`|否|人员唯一编码|
|`++cardNo`|`body`|`string`|否|卡号|
|`++personName`|`body`|`string`|否|人员名称|
|`++orgId`|`body`|`string`|否|组织编码|
|`++orgName`|`body`|`string`|否|组织名称|
|`++deviceName`|`body`|`string`|否|设备名称|
|`++deviceCode`|`body`|`string`|否|设备编码|
|`++receiveTime`|`body`|`string`|否|平台接收事件时间，采用ISO8601时间格式|
|`++deviceName`|`body`|`string`|否|设备名称|
|`++liftName`|`body`|`string`|否|电梯名称|
