# 查询按人员详情与设备下发信息

> 接口说明
> 根据人员或设备通道分页查询人员详情与设备下发信息

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5.0及以上版本|
|接口地址|`/api/acps/v1/authDownload/special/person/diy/result`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/authDownload/special/person/diy/result
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personIds`|`body`|`string[]`|是|人员ID列表，查询人员列表v2接口获取返回报文中的personId字段|
|`resourceInfos`|`body`|`object[]`|是|资源设备对象信息|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的indecCode字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|是|资源通道号|
|`personStatus`|`body`|`integer[]`|否|人员状态|
|`cardStatus`|`body`|`string[]`|否|卡片状态|
|`faceStatus`|`body`|`string[]`|否|人脸状态|
|`pageNo`|`body`|`integer`|否|页码,pageNo>0|
|`pageSize`|`body`|`integer`|否|页大小,0<pageSize<=1000|

## 请求参数示例
```json
{
  "personIds": [
    "8339e26e87b04b009b996275ef68df3a"

    "1"
  ],
  "faceStatus": [
    "1"
  ],
  "pageNo": 1,
  "pageSize": 20
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|总条数|
|`+list`|`body`|`object[]`|否|人员与设备下载信息列表|
|`++personId`|`body`|`string`|否|人员编号|
|`++resourceIndexCode`|`body`|`string`|否|资源的唯一标识|
|`++channelNo`|`body`|`number`|否|通道号|
|`++channelIndexCode`|`body`|`string`|否|资源通道唯一标识（在权限下载和配置接口入参时对应查询接口会返回，不作为查询接口的查询条件）|
|`++personStatus`|`body`|`number`|否|人员状态, 0 已配置未下载, 1 更新待下载, 2 更新待删除, 3 已下载, 4 未配置|
|`++cardStatus`|`body`|`number`|否|卡片状态, 0 已配置未下载, 1 更新待下载, 2 更新待删除, 3 已下载, 4 未配置|
|`++faceStatus`|`body`|`number`|否|人脸状态, 0 已配置未下载, 1 更新待下载, 2 更新待删除, 3 已下载, 4 未配置|
|`++configTime`|`body`|`string`|否|配置时间|
|`++startTime`|`body`|`string`|否|人员权限有效期（以人为中心设备时使用）,开始时间|
|`++endTime`|`body`|`string`|否|人员权限有效期（以人为中心设备时使用）,结束时间|
|`+pageSize`|`body`|`number`|否|页大小|
|`+pageNo`|`body`|`number`|否|页码|
