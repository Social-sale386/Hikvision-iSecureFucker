# 查询权限条目总数

> 接口说明
> 根据人员或设备通道查询权限条目总数。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/auth_item/total/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/auth_item/total/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personIds`|`body`|`string[]`|否|人员ID列表，查询人员列表v2接口获取返回报文中的personId字段|
|`orgIds`|`body`|`string[]`|否|组织ID列表，查询组织列表接口获取返回报文中的orgIndexCode字段|
|`resourceInfos`|`body`|`object[]`|否|资源设备对象信息|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的indexCode字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`string[]`|否|资源通道号|
|`queryType`|`body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`personStatus`|`body`|`integer[]`|否|人员状态|
|`cardStatus`|`body`|`integer[]`|否|卡片状态|
|`faceStatus`|`body`|`integer[]`|否|人脸状态|
|`configTime`|`body`|`object`|否|配置时间查询对象|
|`+startTime`|`Body`|`string`|否|开始日期,采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`+endTime`|`Body`|`string`|否|结束日期,采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`integer`|否|权限条目总数|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": 2
}
```
