# 添加待下载的设备通道_根据出入权限配置下载

> 接口说明
> 选择要下发的设备或点位的所有人权限。
> 如果不调用此接口，代表下发所有人的所有设备和所有资源点的权限。
> 该接口支持向新建的下载任务中添加待下载的权限数据，可通过本接口分多次向下载任务中添加数据。
> 该接口强依赖于资源目录公共存储，请确保设备与人员相关的信息已存在公共存储，否则下载必定失败。
> 数据添加完成后必须调用 开始下载任务接口 才能下发权限。
> 单次接口最多支持100个设备通道对象。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/download/configuration/data/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/download/configuration/data/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`taskId`|`body`|`string`|是|下载任务唯一标识，创建任务处获取|
|`resourceInfos`|`body`|`object[]`|否|资源对象|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的personId字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|资源通道号|

## 请求参数示例
```json
{
  "taskId": "02e66ac47a03481fa0f304fa3acfa773",
  "resourceInfos": [
    {
      "resourceIndexCode": "0a869b06f4ff4da29949acf006405c39",
      "resourceType": "acsDevice",
      "channelNos": [
        1
      ]
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`string`|否|无返回结果|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": "null"
}
```
