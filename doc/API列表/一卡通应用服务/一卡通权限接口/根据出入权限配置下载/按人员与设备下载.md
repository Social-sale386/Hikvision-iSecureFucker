# 按人员与设备下载

> 接口说明
> 接口支持权限配置完成后，下发或重置某个人员或设备的权限。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5.0及以上版本|
|接口地址|`/api/acps/v1/authDownload/configuration/person/reset`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/authDownload/configuration/person/reset
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`resourceInfos`|`body`|`object[]`|否|资源对象列表|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的indexCode字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|资源通道号，|
|`personIds`|`body`|`string[]`|是|人员编号列表|
|`isReset`|`body`|`boolean`|否|是否重置权限(默认只下未下发数据)|

补充说明:
- 当资源类型为设备时，无特殊说明必填。
- 传参数说明：
- 表示此次数据需要配置/下载到设备的哪些通道上；
- 门禁的人证设备、可视对讲设备等无通道的设备，默认通道号为1；

## 请求参数示例
```json
{
  "resourceInfos": [
    {
      "resourceIndexCode": "0a869b06f4ff4da29949acf006405c39",
      "resourceType": "acsDevice",
      "channelNos": [
        1
      ]
    }
  ],
  "personIds": [
    "37835310f786451a8eb55d6c43b827cc"
  ],
  "isReset": false
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success"
}
```
