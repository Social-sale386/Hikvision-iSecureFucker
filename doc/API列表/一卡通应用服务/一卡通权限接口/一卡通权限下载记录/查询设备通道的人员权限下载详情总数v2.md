# 查询设备通道的人员权限下载详情总数v2

> 接口说明
> 根据查询条件查询设备通道的人员下载详情总数，支持下载中和已完成下载的设备通道查询。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/acps/v2/download_record/person/total/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v2/download_record/person/total/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`downloadResultId`|`body`|`string`|否|下载记录唯一ID,使用downloadResultId查询设备通道下载记录详情时，忽略taskId与resourceInfo字段（“下载记录唯一ID”和“下载任务标识+设备通道对象”两者必须有一个不为空），从查询设备通道权限下载记录列表接口获取返回参数downloadResultId|
|`taskId`|`body`|`string`|否|下载任务唯一标识,，从查询设备通道权限下载记录列表接口获取返回参数taskId|
|`resourceInfo`|`body`|`object`|否|设备通道对象列表|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的personId字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|通道号，当资源类型为设备时，无特殊说明必填|
|`personIds`|`body`|`string[]`|否|人员编号列表，查询人员列表v2接口获取返回报文中的personId字段|
|`orgId`|`body`|`string[]`|否|组织编号,查询组织列表接口获取返回报文中的orgIndexCode字段|
|`downloadResult`|`body`|`integer`|否|下载结果，|
|`downloadTime`|`body`|`object`|否|下载时间查询对象|
|`+startTime`|`Body`|`string`|否|开始日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`+endTime`|`Body`|`string`|否|结束日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|

补充说明:
- 0 成功，
- 1 失败，
- 2 部分成功

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`integer`|否|下载详情总数|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": 2
}
```
