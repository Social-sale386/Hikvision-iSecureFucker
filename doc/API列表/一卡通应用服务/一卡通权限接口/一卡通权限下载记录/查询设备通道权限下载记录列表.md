# 查询设备通道权限下载记录列表

> 接口说明
> 根据查询条件分页查询设备通道的下载记录(包括计划模板的下载记录)，只能查询由业务组件自身创建的任务下载记录。下载记录主要展示此次下载的概览信息，可通过查询设备通道的下载记录详情接口查询每个设备通道中人员的下载详情。
> 支持通过任务编号查询单个任务的下载记录。
> 支持通过设备通道查询设备通道的历史下载记录（卡权限下载记录只能通过设备查询，人脸、指纹下载可通过设备和通道查询）。
> 支持通过任务编号、设备通道对象、下载时间、下载类型查询历史下载记录。
> 该接口仅返回分页的列表数据，不返回总数。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/download_record/channel/list/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/download_record/channel/list/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`taskId`|`body`|`string`|否|下载任务唯一标识，创建任务时获得|
|`resourceInfos`|`body`|`object[]`|否|设备通道对象列表|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的personId字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|通道号|
|`taskTypes`|`body`|`integer[]`|否|下载任务类型|
|`downloadResult`|`body`|`integer`|否|下载结果，|
|`downloadStartTime`|`body`|`object`|否|下载开始时间查询对象|
|`+startTime`|`Body`|`string`|否|开始日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`+endTime`|`Body`|`string`|否|结束日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`downloadEndTime`|`body`|`object`|否|下载结束时间查询对象|
|`+startTime`|`Body`|`string`|否|结束日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`+endTime`|`Body`|`string`|否|结束日期，采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`sortObjects`|`body`|`object[]`|否|排序对象，目前只支持下载开始与结束时间排序|
|`+field`|`Body`|`string`|否|排序字段|
|`+sort`|`Body`|`string`|否|排序规则，asc：正序，|
|`pageNo`|`body`|`integer`|是|页码，pageNo>0|
|`pageSize`|`body`|`integer`|是|当前页展示数目，0<pageSize<=1000|

补充说明:
- 1：卡片
- 4：人脸
- 其他详见附录A.53 任务下载类型说明
- 0成功，
- 1失败，
- 2部分成功
- desc：倒序

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|总条数|
|`+list`|`body`|`object[]`|否|设备通道的下载记录|
|`++downloadResultId`|`body`|`string`|否|下载记录唯一ID|
|`++tagId`|`body`|`string`|否|标签|
|`++taskId`|`body`|`string`|否|下载任务唯一标识|
|`++taskOptType`|`body`|`number`|否|任务操作类型|
|`++taskType`|`body`|`number`|否|下载任务类型|
|`++resourceInfo`|`body`|`object`|否|设备通道对象|
|`+++resourceIndexCode`|`body`|`string`|否|资源的唯一标识|
|`+++resourceType`|`body`|`string`|否|资源类型，详见附录A.50 设备通道类型|
|`+++channelNos`|`body`|`number[]`|否|资源通道号|
|`+++channelIndexCodes`|`body`|`string[]`|否|资源通道唯一标识（在权限下载和配置接口入参时对应查询接口会返回，不作为查询接口的查询条件）|
|`++downloadResult`|`body`|`number`|否|下载结果,|
|`++errorCode`|`body`|`string`|否|下载错误码,在出现异常时有值（比如：设备不在线导致的下载失败，设备能力集不支持，驱动异常，服务异常等）|
|`++downloadPersonCount`|`body`|`number`|否|已下发人员数量|
|`++successedPersonCount`|`body`|`number`|否|下载成功人员数量|
|`++failedPersonCount`|`body`|`number`|否|下载失败人员数量|
|`++taskOptType`|`body`|`number`|否|任务操作类型|
|`++startTime`|`body`|`string`|否|下载开始时间，采用ISO8601时间格式，如2019-09-03T17:30:08.000+08:00|
|`++endTime`|`body`|`string`|否|下载结束时间，采用ISO8601时间格式，如2019-09-03T17:30:08.000+08:00|

补充说明:
- （0:初始化下载；
- 1:异动下载；
- 2:指定下载；
- 3:快速下载；
- 4:自动下载；
- 5:同步下载）
- 1：卡片
- 4：人脸
- 其他详见附录A.53 任务下载类型说明
- 0成功,
- 1失败,
- 2部分成功
- 0:初始化下载；1:异动下载；
- 2:指定下载；
- 3:快速下载；
- 4:自动下载；
- 5:同步下载
