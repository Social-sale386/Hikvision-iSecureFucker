# 添加权限配置

> 接口说明
> 权限配置支持按组织、人员和设备通道配置权限，适用综合大楼、学校、医院等场景。
> 说明：权限配置数据采用异步分批入库方式，接口调用成功后返回权限配置单编号，在配置的过程中分批插入数据，只有当配置单结束时才能查询到完整的权限配置信息。相同的配置数据重复配置时，第一次配置生效后，后面相同的配置将自动过滤丢弃。
> 合作方配置的tagId用于让多个应用共用出入控制权限服务时，用以区分各自的配置信息。
> 注意点：不同业务组件设备通道隔离，应该根据业务场景使用不同的设备通道配置权限；如对相同的设备通道都有业务应用，那么人员隔离，应该根据场景使用不同的人员，否则会造成权限条目归属相互竞争的情况，在权限条目综合查询时，数据归属以最后一次入库配置为准。
> 单次接口最多支持1000个设备资源和1000个人员数据。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/auth_config/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/auth_config/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personDatas`|`body`|`object[]`|是|人员数据列表|
|`+indexCodes`|`Body`|`string[]`|是|该参数和personDataType组合使用,|
|`+personDataType`|`Body`|`string`|是|人员数据类型，|
|`resourceInfos`|`body`|`object[]`|否|设备通道对象列表，最大1000条|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的indexCode字段；|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|通道号|
|`startTime`|`body`|`string`|否|开始日期，开始与结束日期要么都为空要么都不为空。配置人员权限的有效期，为空时默认长期有效。采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|
|`endTime`|`body`|`string`|否|结束日期，开始与结束日期要么都为空要么都不为空。配置人员权限的有效期，为空时默认长期有效。采用ISO8601时间格式，最大长度32个字符，如2018-09-03T17:30:08+08:00|

补充说明:
- 当personDataType传入组织时,indexCodes传入的是组织ID，获取组织列表接口获取返回报文中的orgIndexCode字段；
- 当personDataType传入人员时,indexCodes传入的是人员ID，获取人员列表v2接口获取返回报文中的personId字段；最大1000条
- person人员，
- org组织

## 请求参数示例
```json
{
  "personDatas": [
    {
      "indexCodes": [
        "c35f4a4e-8baa-4e88-b6e3-c4060135e7e6"
      ],
      "personDataType": "person"
    }
  ],
  "resourceInfos": [
    {
      "resourceIndexCode": "0a869b06f4ff4da29949acf006405c39",
      "resourceType": "acsDevice",
      "channelNos": [
        1
      ]
    }
  ],
  "startTime": "2018-09-03T17:30:08+08:00",
  "endTime": "2018-09-06T17:30:08+08:00"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+taskId`|`body`|`string`|否|配置单编号|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "taskId": "317735290e9047898f6bb2f9aa0ec745"
  }
}
```
