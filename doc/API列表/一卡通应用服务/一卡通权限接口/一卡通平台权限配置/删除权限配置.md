# 删除权限配置

> 接口说明
> 根据人员数据、设备通道删除已配置的权限，合作方配置的tagId用于让多个应用共用出入控制权限服务时，用以区分各自的配置信息，即只能删除同一个tagId的权限配置信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/auth_config/delete`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/auth_config/delete
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personDatas`|`body`|`object[]`|否|人员数据列表|
|`+indexCodes`|`Body`|`string[]`|是|人员数据编号|
|`+personDataType`|`Body`|`string`|是|人员数据类型， person人员， org 组织|
|`resourceInfos`|`body`|`object[]`|否|设备通道对象列表|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中indexCode字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|通道号|

## 请求参数示例
```json
{
  "personDatas": [
    {
      "indexCodes": [
        "59da91f7-0185-4cd2-9f72-013c1243d2f5"
      ],
      "personDataType": "personGroup"
    }
  ],
  "resourceInfos": [
    {
      "resourceIndexCode": "0a869b06f4ff4da29949acf006405c39",
      "resourceType": "acsDevice",
      "channelNos": [
        1
      ]
    }
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+taskId`|`body`|`string`|否|配置单编号|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "taskId": "317735290e9047898f6bb2f9aa0ec745"
  }
}
```
