# 人员权限使用状态设置

> 接口说明
> 设置人员权限的使用状态，默认人员权限使用状态为有效，当设置其他状态后可通过设置权限有效状态恢复。
> 说明：针对智能锁设备，人员权限状态的禁用和有效对应智能锁的冻结解冻功能。
> 按权限条目（人员与设备通道）设置可以通过权限条目综合查询章节接口可获取人员的权限使用状态。
> 按人员全局设置可以通过查询人员权限全局状态设置接口获取人员列表

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5.0及以上版本|
|接口地址|`/api/acps/v1/authConfig/personAuth/useStatus/configuration`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/authConfig/personAuth/useStatus/configuration
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personIds`|`body`|`string[]`|是|人员ID列表，查询人员列表v2接口获取返回报文中的personId字段|
|`useStatus`|`body`|`integer`|是|权限使用状态,|
|`resourceInfos`|`body`|`object[]`|否|资源设备对象列表，当为空时，表示人员权限全局设置|
|`+resourceIndexCode`|`Body`|`string`|否|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的personId字段|
|`+resourceType`|`Body`|`string`|否|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|否|资源通道号|

补充说明:
- 0-权限有效，
- 1-权限禁用

## 请求参数示例
```json
{
  "personIds": [
    "37835310-f786-451a-8eb5-5d6c43b827cc"
  ],
  "useStatus": 1
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`string`|否|无返回结果|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success"
}
```
