# 查询权限配置

> 接口说明
> 根据人员数据（组织、人员）、设备通道查询已配置的权限，合作方配置的tagId用于让多个应用共用出入控制权限服务时，用以区分各自的配置信息，即只能查询到同一个tagId的权限配置信息。
> 接口中人员数据、设备通道、设备通道分组至少一个不为空。
> acpsV1.5.0版本查询和返回字段新增personExtendData，仅在按人员配置了人员权限扩展参数的情况下才有值。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/acps/v1/auth_config/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/acps/v1/auth_config/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`personDataIds`|`body`|`string[]`|否|人员数据编号|
|`personDataType`|`body`|`string`|否|人员数据类型， person人员， org 组织|
|`resourceInfos`|`body`|`object[]`|否|设备通道对象列表|
|`+resourceIndexCode`|`Body`|`string`|是|资源的唯一标识，资源为设备，查询资源列表v2接口获取返回报文中的personId字段|
|`+resourceType`|`Body`|`string`|是|资源类型，详见附录A.50 设备通道类型|
|`+channelNos`|`Body`|`number[]`|是|通道号|
|`resourceDataType`|`body`|`string`|否|查询资源类型，|
|`pageNo`|`body`|`integer`|是|页码，pageNo>0|
|`pageSize`|`body`|`integer`|是|页大小，0<pageSize<=1000|

补充说明:
- resource 设备通道，
- resourceGroup 设备通道分组

## 请求参数示例
```json
{
  "personDataIds": [
    "59da91f7-0185-4cd2-9f72-013c1243d2f5"
  ],
  "personDataType": "person",

      "channelNos": [
        1
      ]
    }
  ],
  "resourceDataType": "resource",
  "pageNo": 1,
  "pageSize": 20
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.2.3 出入控制权限错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|总条数|
|`+pageSize`|`body`|`number`|否|页大小|
|`+pageNo`|`body`|`number`|否|页码|
|`+list`|`body`|`object[]`|否|权限配置信息列表|
|`++personDataId`|`body`|`string`|否|人员数据编号|
|`++personDataType`|`body`|`string`|否|人员数据类型， person 人员， org 组织|
|`++resourceIndexCode`|`body`|`string`|否|资源数据编号，resourceDataType为设备通道时值为设备indexcode；resourceDataType为设备通道分组时值为设备通道分组编号；|
|`++resourceType`|`body`|`string`|否|资源类型，详见附录A.50 设备通道类型|
|`++resourceDataType`|`body`|`string`|否|资源数据类型，resource 设备通道， resourceGroup 设备通道分组|
|`++channelNo`|`body`|`number`|否|通道号|
|`++channelIndexCode`|`body`|`string`|否|资源通道唯一标识（在权限下载和配置接口入参时对应查询接口会返回，不作为查询接口的查询条件）|
|`++templateId`|`body`|`string`|否|计划模板编号，为空时，默认为全天候计划模板。|
|`++startTime`|`body`|`string`|否|开始日期|
|`++endTime`|`body`|`string`|否|结束日期|
|`++personExtendData`|`body`|`object`|否|人员扩展参数|
|`+++personPropertypersonProperty`|`body`|`number`|否|人员特性， 1：普通人，默认类型； 2：来宾，来宾人员比对通过或者时，会上报来宾事件，可正常开门； 2：黑名单，黑名单人员比对通过时，会上报黑名单事件，且无法开门； 4:超级用户，为需要超级权限的用户而设置。不受限于门常闭、反潜回、刷卡+密码认证需要密码确认的规则，刷卡直接开门，但要遵循多门互锁及多重认证规则。|
|`+++userVerifyMode`|`body`|`string`|否|人员验证方式， 人员验证方式的优先级高于读卡器验证方式，需要根据设备能力集判断，设备仅支持部分字段）：cardAndPw-刷卡+密码，card-刷卡，cardOrPw-刷卡或密码，fp-指纹，fpAndPw-指纹+密码，fpOrCard-指纹或刷卡，fpAndCard-指纹+刷卡，fpAndCardAndPw-指纹+刷卡+密码，faceOrFpOrCardOrPw-人脸或指纹或刷卡或密码，faceAndFp-人脸+指纹，faceAndPw-人脸+密码，faceAndCard-人脸+刷卡，face-人脸， fpOrPw-指纹或密码， faceAndFpAndCard-人脸+指纹+刷卡，faceAndPwAndFp-人脸+密码+指纹， faceOrfaceAndCard-人脸或人脸+刷卡，fpOrface-指纹或人脸，cardOrfaceOrPw-刷卡或人脸或密码，cardOrFace-刷卡或人脸，cardOrFaceOrFp刷卡或人脸或指纹（默认设备本身的认证方式，仅支持以人为中心设备）|
|`+++maxOpenDoorTime`|`body`|`number`|否|最大认证次数，当且仅当personProperty是来宾时有效 0为无次数限制（默认为0）|
|`+++leaderPersonEnabled`|`body`|`boolean`|否|是否是首人，需要在设备上配置首卡开门持续时长|
|`+++closeDelayEnabled`|`body`|`boolean`|否|是否关闭延迟开门，需在设备上配置延迟开门时长|
