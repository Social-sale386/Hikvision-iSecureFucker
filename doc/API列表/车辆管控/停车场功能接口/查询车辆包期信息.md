# 查询车辆包期信息

> 接口说明
> 简述：车辆包期后在当前停车场是固定车，自由进出场；在未包期的停车场进出场是临时车，需要收费。可通过此接口查询平台所有车辆或某个停车场里车辆的包期状态，便于展示车辆包期状态和是否固定车查询。
> 支持：支持通过车牌号、停车场编号分页查询车辆包期信息。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/pms/v1/car/charge/page`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/pms/v1/car/charge/page
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`parkSyscodes`|`body`|`string`|否|停车库唯一标识集合，可从获取停车库列表接口获取返回参数parkIndexCode|
|`plateNo`|`body`|`string`|否|车牌号|
|`pageNo`|`body`|`integer`|是|目标页码，0 < pageNo|
|`pageSize`|`body`|`integer`|是|每页记录数，0 < pageSize≤1000|
|`cardNo`|`body`|`string`|否|卡号|
|`regionIndexCode`|`body`|`string`|否|区域IndexCode，根据查询区域列表v2接口获取返回报文中的indexCode字段|

## 请求参数示例
```json
{
  "parkSyscodes": "dfasgqegbq34g3q4g34g",
  "plateNo": "浙A12345",
  "pageNo": 1,
  "pageSize": 15,
  "cardNo": "1234567890",
  "regionIndexCode": "root00000"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.3.1 停车场应用错误码|
|`msg`|`body`|`string`|否|返回描述-记录接口执行情况说明信息|
|`data`|`body`|`object`|否|返回数据-车辆包期信息分页结构|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|每页记录总数|
|`+list`|`body`|`object[]`|否|车辆对象列表|
|`++regionIndexCode`|`body`|`string`|否|区域编号|
|`++vehicleId`|`body`|`string`|否|车辆编号|
|`++plateNo`|`body`|`string`|否|车牌号码|
|`++cardNo`|`body`|`string`|否|卡号，绑定了卡号才有此信息|
|`++personId`|`body`|`string`|否|车主编号|
|`++personName`|`body`|`string`|否|车主，绑定了车主才有此信息|
|`++categoryCode`|`body`|`string`|否|车辆分类标识|
|`++categoryName`|`body`|`string`|否|车辆分类名称|
|`++groupName`|`body`|`string`|否|群组名称，配置了群组才有此信息|
|`++validity`|`body`|`object[]`|否|权限信息列表，包期充值了才有此消息|
|`+++parkSyscode`|`body`|`string`|否|停车库唯一标识|
|`+++parkName`|`body`|`string`|否|停车库名称|
|`+++functionTime`|`body`|`object[]`|否|有效期信息，当前停车库有效期集合|
|`++++startTime`|`body`|`string`|否|开始时间（时间精确到天，格式：yyyy-MM-dd）|
|`++++endTime`|`body`|`string`|否|结束时间（时间精确到天，格式：yyyy-MM-dd）|
