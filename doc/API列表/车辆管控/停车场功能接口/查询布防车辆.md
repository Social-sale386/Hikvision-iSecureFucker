# 查询布防车辆

> 接口说明
> 简述：布防车辆即平台黑名单车辆信息，用于车辆进出场管控；可通过此接口查询平台布防车辆信息。
> 支持：支持通过按照车牌、卡号查询布防车辆信息，支持分页。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/pms/v1/alarmCar/page`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/pms/v1/alarmCar/page
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`searchKey`|`body`|`string`|否|车牌号/卡号|
|`pageNo`|`body`|`integer`|是|目标页码，范围(0,~)|
|`pageSize`|`body`|`integer`|是|每页记录数，范围(0,1000]|
|`startTime`|`body`|`string`|否|查询开始时间（格式：yyyy-MM-ddTHH:mm:ss+当前时区）|
|`endTime`|`body`|`string`|否|查询结束时间（格式：yyyy-MM-ddTHH:mm:ss+当前时区）|
|`regionIndexCode`|`body`|`string`|否|区域IndexCode，根据查询区域列表v2接口获取返回报文中的indexCode字段|
|`parkingIndexCode`|`body`|`string`|是|停车场编号,范围[0,64]，只查询该停车库下的信息|

## 请求参数示例
```json
{
  "searchKey": "浙A12345",
  "pageNo": 1,
  "pageSize": 15,
  "startTime": "2018-07-26T15:00:00+08:00",
  "endTime": "2018-07-26T17:00:00+08:00",
  "regionIndexCode": "root00000",
  "parkingIndexCode": "11111111111"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.3.1 停车场应用错误码|
|`msg`|`body`|`string`|否|返回描述-记录接口执行情况说明信息|
|`data`|`body`|`object`|否|返回数据-布防车辆分页结构|
|`+total`|`body`|`number`|否|查询数据记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|每页记录总数|
|`+list`|`body`|`object[]`|否|布防车辆对象列表|
|`++alarmSyscode`|`body`|`string`|否|布防唯一标识|
|`++plateNo`|`body`|`string`|否|车牌号码|
|`++cardNo`|`body`|`string`|否|卡号|
|`++driver`|`body`|`string`|否|驾驶员名称|
|`++driverPhone`|`body`|`string`|否|驾驶员电话|
|`++remark`|`body`|`string`|否|备注信息|
|`++endTime`|`body`|`string`|否|布防结束时间,ISO8601格式：|

补充说明:
- yyyy-MM-ddTHH:mm:ss+当前时区，例如北京时间：
- 2018-07-26T15:00:00+08:00
