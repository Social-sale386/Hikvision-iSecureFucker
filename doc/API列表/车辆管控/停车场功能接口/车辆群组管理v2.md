# 车辆群组管理v2

> 接口说明
> 简述：常用车辆分类信息有固定车、临时车、预约车、特殊车、黑名单，以及各种车辆群组。
> 一个区域下一个车辆只能绑定一个群组类型。
> 支持：将车辆信息和群组进行增量绑定或取消绑定，用作相关业务逻辑处理
> 说明：临时车:系统车辆默认是临时车;
> 黑名单：通过【车辆布防接口】添加;
> 固定车：通过【车辆充值接口】添加;
> 预约车：通过【车位预约接口】添加;
> 群组车：通过本接口添加;
> 特殊车：暂不支持第三方配置。

## 基本信息
|项目|值|
|---|---|
|接口版本|v2|
|适配版本|综合安防管理平台iSecure Center V1.6及以上版本|
|接口地址|`/api/pms/v2/car/categoryBind`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/pms/v2/car/categoryBind
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`vehicleIds`|`body`|`string`|是|车辆标识集合（以‘,’分割），可从查询车辆列表v2接口获取返回参数vehicleId|
|`categoryCode`|`body`|`string`|否|车辆分类标识，可从查询车辆分类接口获取返回参数categoryCode，|
|`regionIndexCode`|`body`|`string`|否|区域IndexCode，根据查询区域列表v2接口获取返回报文中的indexCode字段|
|`operation`|`body`|`integer`|是|绑定操作类型，|

补充说明:
- 默认分类以外是群组分类,默认分类参考附录A.80 默认车辆分类，绑定的时候必填,取消绑定的时候和区域二选一
- 1 绑定
- 2 解绑

## 请求参数示例
```json
{
  "vehicleIds": "19affb87-7a2a-4b3b-9744-7308708618ea,f6dedacb-8d22-461c-a924-4219e398c9ac",
  "categoryCode": "9",
  "regionIndexCode": "19affb11-7a2a-4b3b-9744-7308708618ea",
  "operation": 1
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.3.1 停车场应用错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+vehicleInfoRespInfoList`|`body`|`object[]`|否|车辆返回结果信息|
|`++regionIndexCode`|`body`|`string`|否|区域编号|
|`++vehicleId`|`body`|`string`|否|车辆编号|
|`++plateNo`|`body`|`string`|否|车牌号|
|`++cardNo`|`body`|`string`|否|卡号|
|`++personId`|`body`|`string`|否|车主编号|
|`++personName`|`body`|`string`|否|车主名称|
|`++categoryCode`|`body`|`string`|否|车辆分类标识,默认分类以外是群组分类,默认分类参考附录A.80 默认车辆分类|
|`++categoryName`|`body`|`string`|否|车辆分类名称|
