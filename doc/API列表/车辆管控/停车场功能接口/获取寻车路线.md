# 获取寻车路线

> 接口说明
> 简述：路线规划需要起点信息和终点信息，根据起始点编号获取到两点在地图上经纬度信息，从而规划出两点间的路线，此接口提供路线展示的页面信息。
> 支持：支持通过起始点车位标号获取两点间的路线信息的页面。
> 支持多线路/网域过滤。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.2及以上版本|
|接口地址|`/api/pms/v1/map/vehicleRoute`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/pms/v1/map/vehicleRoute
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`originFloorSyscode`|`body`|`string`|是|起点楼层编号，可从查询停车库节点信息接口获取返回参数中楼层的indexCode|
|`originLong`|`body`|`string`|是|起点经度|
|`originLat`|`body`|`string`|是|起点维度|
|`endSpaceSyscode`|`body`|`string`|是|终点车位编号，可从查询停车库节点信息接口获取返回参数中车位的indexCode|

## 请求参数示例
```json
{
  "originFloorSyscode": "dfasgqegbq34g3q4g34g",
  "originLong": "120.52135613131",
  "originLat": "30.5623841266523",
  "endSpaceSyscode": "ddsrewr43253g35y2tg5"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.3.1 停车场应用错误码|
|`msg`|`body`|`string`|否|返回描述-记录接口执行情况说明信息|
|`data`|`body`|`string`|否|为空|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": null
}
```
