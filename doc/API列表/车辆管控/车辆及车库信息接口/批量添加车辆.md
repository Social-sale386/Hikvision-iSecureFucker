# 批量添加车辆

> 接口说明
> 单个添加车辆信息接口，注意，车辆的必选字段必须都包括在入参中。
> 若需支持批量添加的后续业务处理，请求需指定每个车辆的clientId，再通过返回值中的clientId判断哪些成功，哪些失败。所以建议每次接口调用，clientid保持唯一。
> 注：clientid只对本次调用有效

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/vehicle/batch/add`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/vehicle/batch/add
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`Array[0]`|`body`|`object`|是|待添加的车辆列表，单次提交上限为1000条记录|
|`+clientId`|`Body`|`number`|是|调用方指定标识，接口执行成功后将服务端生成的标识与此标识绑定后返回，再通过返回值中的clientId判断哪些成功，哪些失败。所以建议每次接口调用，clientid保持唯一。 注：clientid只对本次调用有效|
|`+plateNo`|`Body`|`string`|是|车牌号码|
|`+personId`|`Body`|`string`|否|人员ID|
|`+plateType`|`Body`|`string`|否|车牌类型，详见附录A.14 车牌类型|
|`+plateColor`|`Body`|`string`|否|车牌颜色，详见附录A.15 车牌颜色|
|`+vehicleType`|`Body`|`string`|否|车辆类型，详见附录A.16 车辆类型|
|`+vehicleColor`|`Body`|`string`|否|车辆颜色，详见附录A.17 车辆颜色|
|`+description`|`Body`|`string`|否|车辆描述|

## 请求参数示例
```json
[
  {
    "clientId": 1,
    "plateNo": "浙A111111",
    "personId": "26v25v62562v65727v27",
    "plateType": "1",
    "plateColor": "1",
    "vehicleType": "0",
    "vehicleColor": "0",
    "description": "test"
  }
]
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回结果|
|`+successes`|`body`|`object[]`|否|批量添加成功的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++vehicleId`|`body`|`string`|否|服务端生成的唯一标识|
|`+failures`|`body`|`object[]`|否|批量添加失败的记录|
|`++clientId`|`body`|`number`|否|调用方指定Id|
|`++code`|`body`|`string`|否|错误码|
|`++msg`|`body`|`string`|否|错误信息|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "successes": [
      {
        "clientId": 123456789,
        "vehicleId": "5668f44c-ae23-4674-9b79-d8f79f77fd41"
      }
    ],
    "failures": [
      {
        "clientId": 555,
        "code": "-1",
        "msg": "error"
      }
    ]
  }
}
```
