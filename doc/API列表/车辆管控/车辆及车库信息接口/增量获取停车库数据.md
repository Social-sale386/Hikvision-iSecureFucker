# 增量获取停车库数据

> 接口说明
> 根据查询条件查询停车库列表信息，主要根据时间段分页获取停车库信息，包含已删除数据。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/resource/v1/park/timeRange`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/park/timeRange
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`startTime`|`body`|`string`|是|开始日期，要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00表示北京时间2017年7月26日21时30分08秒|
|`endTime`|`body`|`string`|否|截止日期，要求遵守ISO8601标准，如2018-07-26T21:30:08.000+08:00表示北京时间2017年7月26日21时30分08秒，不传时默认为当前时间|
|`pageNo`|`body`|`integer`|是|页码,pageNo≥1|
|`pageSize`|`body`|`integer`|是|页面大小,0<pageSize≤1000|
|`resourceType`|`body`|`string`|是|资源类型|

补充说明:
- 停车库:parking
- 出入口：entrance
- 车道：roadway
- 楼层：parkFloor
- 车位：parkspace

## 请求参数示例
```json
{
  "startTime": "2018-07-26T21:30:08.000+08:00",
  "endTime": "2018-07-26T23:30:08.000+08:00",
  "pageNo": 1,
  "pageSize": 1000,
  "resourceType": "parking"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|返回信息|
|`+total`|`body`|`number`|否|结果总数|
|`+list`|`body`|`object[]`|否|返回数据列表|
|`++indexCode`|`body`|`string`|否|唯一标识|
|`++name`|`body`|`string`|否|名称|
|`++resourceType`|`body`|`string`|否|资源类型|
|`++parentIndexCode`|`body`|`string`|否|父节点唯一标识|
|`++status`|`body`|`number`|否|状态,0：有效的 ；小于0：删除的，如-1|
|`++updateTime`|`body`|`string`|否|更新日期，遵守ISO8601标准，如2018-07-26T21:30:08+08:00表示北京时间2017年7月26日21时30分08秒|
|`++createTime`|`body`|`string`|否|创建日期，遵守ISO8601标准，如2018-07-26T21:30:08+08:00表示北京时间2017年7月26日21时30分08秒|
|`++regionIndexCode`|`body`|`string`|否|所属区域ID|
|`++parkPath`|`body`|`string`|否|停车场资源indexCode组成的目录，以@分隔|

补充说明:
- 停车库:parking
- 出入口：entrance
- 车道：roadway
- 楼层：parkFloor
- 车位：parkspace
