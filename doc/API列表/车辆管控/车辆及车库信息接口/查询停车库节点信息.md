# 查询停车库节点信息

> 接口说明
> 查询停车库节点信息，支持同时查询多种类型的节点，用于异步展示停车、搜索等场景。
> 示例：
> 当parentIndexCode、parentResourceType传”123”，” parking”,时，resourceTypes传parking时返回停车库”123”下面的子库信息；
> 当parentIndexCode、parentResourceType传”123”，” parking”,时，resourceTypes传entrance时，返回停车库”123”下面的出入口；
> 当parentIndexCode、parentResourceType传”123”，” parking”,时，resourceTypes传parkFloor时，返回停车库”123”下面的楼层信息；
> 当parentIndexCode、parentResourceType传”123”，” parking”,时，resourceTypes传parking 、entrance、parkFloor时，同时返回停车库”123”下面的子库、出入口、楼层；
> 当parentIndexCode、parentResourceType传”456”，” entrance”,时，resourceTypes传rodaway返回出入口”456”下面的车道

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.3及以上版本|
|接口地址|`/api/resource/v1/park/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/park/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`parentIndexCode`|`body`|`string`|否|父节点编号|
|`parentResourceType`|`body`|`string`|否|父节点类型,parentIndexCode、|
|`resourceTypes`|`body`|`string[]`|否|类型之间为或的关系|
|`name`|`body`|`string`|否|节点名称,模糊匹配|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|是|资源属性名,key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|是|操作运算符,|
|`+values`|`Body`|`string[]`|是|资源属性值,=、>=、<=、like、values数组长度只能是1；|
|`orderBy`|`body`|`string`|否|排序字段,注意：排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序升序,降序：desc|
|`pageSize`|`body`|`integer`|是|分页大小,0<pageSize≤1000|
|`pageNo`|`body`|`integer`|是|当前页码,pageNo≥1|

补充说明:
- parentResourceTypes
- 必须同时存在或者同时不填
- 停车库:parking
- 出入口：entrance
- 车道：roadway
- 楼层：parkFloor
- 车位：parkSpace
- 0 ：=;
- 1 ：>=;
- 2 ：<=;
- 3 ：in;
- 4 ：not in;
- 5 ：between;
- 6 ：like
- in、not in，values数组长度大于1,in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到,between只能用于整形、日期,like只能用于字符串
- 升序：asc

## 请求参数示例
```json
{
  "parentIndexCode": "123",
  "parentResourceType": "parking",
  "resourceTypes": [
    "parking"
  ],
  "name": "11",
  "orderBy": "name",
  "orderType": "desc",
  "pageSize": 1,
  "pageNo": 100
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0:接口业务处理成功 其他-失败|
|`msg`|`body`|`string`|否|接口执行情况说明信息|
|`data`|`body`|`object`|否|返回信息|
|`+total`|`body`|`number`|否|记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|分页大小|
|`+list`|`body`|`object[]`|否|停车库信息结构体|
|`++indexCode`|`body`|`string`|否|唯一标示|
|`++name`|`body`|`string`|否|名称|
|`++resourceType`|`body`|`string`|否|资源类型|
|`++parentIndexCode`|`body`|`string`|否|父节点唯一标示|
|`++parentResourceType`|`body`|`string`|否|父节点类型|
|`++updateTime`|`body`|`string`|否|更新时间，遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|
|`++createTime`|`body`|`string`|否|创建时间，遵守ISO8601标准，如2018-07-26T21:30:08.322+08:00|

|`++parkNamePath`|`body`|`string`|否|停车场资源名称组成的目录，以/分隔|
|`++regionIndexCode`|`body`|`string`|否|区域唯一标示|
|`++regionName`|`body`|`string`|否|区域名称|
|`++regionPath`|`body`|`string`|否|所属区域目录，由唯一标示组成，最大10级，格式： @根节点@子区域1@子区域2@|
|`++regionPathName`|`body`|`string`|否|区域目录名称|

补充说明:
- 停车库:parking
- 出入口：entrance
- 车道：roadway
- 楼层：parkFloor
- 车位：parkSpace
