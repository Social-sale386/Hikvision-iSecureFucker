# 查询车载设备列表

> 接口说明
> 查询目录下有权限的车载设备列表。
> 非必填入参若不传值则使用默认值作为查询限制条件，没有默认值的若不传值则不作为查询限制条件。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/resource/v1/encodeDeviceMss/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/resource/v1/encodeDeviceMss/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`name`|`body`|`string`|否|名称，模糊搜索，最大长度32，若包含中文，最大长度指不超过按照指定编码的字节长度，即getBytes(“utf-8”).length|
|`regionIndexCodes`|`body`|`string[]`|否|区域编号,可以为空;|
|`isSubRegion`|`body`|`boolean`|否|true时，搜索regionIndexCodes及其子孙区域的资源，false时，只搜索 regionIndexCodes的资源，不传默认为false|
|`pageNo`|`body`|`integer`|是|当前页码|
|`pageSize`|`body`|`integer`|是|分页大小|
|`expressions`|`body`|`object[]`|否|查询表达式|
|`+key`|`Body`|`string`|否|资源属性名，key传updateTime，operator传between可以查询特定时间段更新的数据，考虑到校时和夏令时，建议值查询过去一天的数据变更|
|`+operator`|`Body`|`number`|否|操作运算符， 0 ：=， 1 ：>=， 2 ：<=， 3 ：in， 4 ：not in， 5 ：between， 6 ：like， 7 ：pre like， 8 ：suffix like|
|`+values`|`Body`|`string[]`|否|资源属性值，=、>=、<=、like、values数组长度只能是1； in、not in，values数组长度大于1，最大不超时20； in_array用于查询key值有多个value的情况，例如行车监控添加的设备类型为encodeDevice、encodeDeviceMss两个类型，使用encodeDevice或者encodeDeviceMss都可以查询到； between只能用于整形、日期 ； like只能用于字符串。|
|`orderBy`|`body`|`string`|否|排序字段,注意：排序字段必须是查询条件，否则返回参数错误|
|`orderType`|`body`|`string`|否|降序升序,降序：desc 升序：asc|

补充说明:
- String[]，支持根据区域批量查询；
- 区域编号个数
- <=1000个；
- 单个长度<=64Byte；，可从查询区域列表v2接口获取返回参数indexCode

## 请求参数示例
```json
{
  "name": "test",
  "regionIndexCodes": [
    "8fyc8qw8280y0y43"
  ],
  "isSubRegion": true,
  "pageNo": 1,
  "pageSize": 1,
  "expressions": [
    {
      "key": "indexCode",
      "operator": 0,
      "values": [
        "ayd8y80y1y082ye01y2e8"
      ]
    }
  ],
  "orderBy": "name",
  "orderType": "desc"
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.other.1 资源目录错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+total`|`body`|`number`|否|记录总数|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|分页大小|
|`+list`|`body`|`object[]`|否|资源数据列表|
|`++deviceKey`|`body`|`string`|否|设备驱动|
|`++indexCode`|`body`|`string`|否|资源唯一编码|
|`++manufacturer`|`body`|`string`|否|厂商|
|`++name`|`body`|`string`|否|资源名称|
|`++netZoneId`|`body`|`string`|否|网域|
|`++port`|`body`|`string`|否|端口|
|`++regionIndexCode`|`body`|`string`|否|所属区域indexCode|
|`++resourceType`|`body`|`string`|否|资源类型，encodeDeviceMss：车载设备，encodeDevice：编码设备|
|`++treatyType`|`body`|`string`|否|接入协议，详见附录A.6 编码设备接入协议|
|`++createTime`|`body`|`string`|否|创建时间，ISO8601标准|
|`++updateTime`|`body`|`string`|否|更新时间，ISO8601标准|
|`++dataVersion`|`body`|`string`|否|版本号|
|`++description`|`body`|`string`|否|描述|
|`++deviceKeyVersion`|`body`|`string`|否|设备驱动版本号|
|`++regionName`|`body`|`string`|否|区域名称|
|`++regionPath`|`body`|`string`|否|区域完整目录，@进行分割|
|`++regionPathName`|`body`|`string`|否|区域目录名称，/进行分割|
