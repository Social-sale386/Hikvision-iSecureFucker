# 查询设备的行驶路线

> 接口说明
> 该接口通过设备编码集合，开始时间，结束时间，用户等条件组合查询三个月内的车载设备行驶路线。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/mss/v1/gps/track/search`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/mss/v1/gps/track/search
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`pageNo`|`body`|`integer`|是|页码，大于0|
|`pageSize`|`body`|`integer`|是|每页数据记录数，范围：[1,1000]|
|`startTime`|`body`|`string`|是|路线查询的开始时间,三个月以内，格式：yyyy-MM-ddTHH:mm:ss+当前时区|
|`endTime`|`body`|`string`|是|路线查询的结束时间，最多查询一天24小时的路线，间距最多为24小时，格式：yyyy-MM-ddTHH:mm:ss+当前时区|
|`deviceIndexCodes`|`body`|`string[]`|是|需要查询路线的设备ID集合，最大1000个|

## 请求参数示例
```json
{
  "pageNo": 1,
  "pageSize": 1000,
  "startTime": "2019-01-01T00:00:00+08:00",
  "endTime": "2019-01-01T00:00:00+08:00",
  "deviceIndexCodes": [
    "d9b1887e8e814a7d86648228cdea8150"
  ]
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.3 行车监控错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+pageNo`|`body`|`number`|否|当前页码|
|`+pageSize`|`body`|`number`|否|每页记录数|
|`+total`|`body`|`number`|否|总条数|
|`+totalPage`|`body`|`number`|否|总页数|
|`+list`|`body`|`object[]`|否|路线信息对象列表|
|`++longitude`|`body`|`string`|否|经度，设备上报的原始坐标，会转成火星坐标（GCJ），然后再转成墨卡托|
|`++latitude`|`body`|`string`|否|纬度，设备上报的原始坐标，会转成火星坐标（GCJ），然后再转成墨卡托|
|`++timeStr`|`body`|`string`|否|gps上报的时间，ISO8601时间格式，格式：yyyy-MM-ddTHH:mm:ss.SSS+当前时区|
|`++speed`|`body`|`string`|否|车辆当前速度，单位：km/h，范围：[0,999]|
|`++deviceIndexCode`|`body`|`string`|否|设备编码|

```json
        "deviceIndexCode": "d9b1887e8e814a7d86648228cdea8150"
      }
    ]
  }
}
```
