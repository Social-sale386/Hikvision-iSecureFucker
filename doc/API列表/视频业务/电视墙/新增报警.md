# 新增报警

> 接口说明
> 1.添加解码器和电视墙。
> 2.关联输出通道至电视墙上。
> 3.窗口已存在且已设置为报警窗口
> 4.调用该接口进行报警上墙

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/tvms/v1/public/tvwall/alarm/addition`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/tvms/v1/public/tvwall/alarm/addition
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`event_name`|`body`|`string`|否|报警事件名称|
|`event_index`|`body`|`string`|是|报警事件编号|
|`dlp_indexcode`|`body`|`string`|是|电视墙indexcode，报警上到哪个电视墙上,从获取电视墙大屏信息接口获取返回参数大屏的index_code|
|`event_level`|`body`|`integer`|是|报警等级，范围：-1～3（1～3分别表示低中高,-1表示无等级，0为扩展值，暂时无效）|
|`event_keep_time`|`body`|`integer`|否|报警超时时间，单位秒。电视墙默认报警超时时间35秒，大于等于5s|
|`alarm_stream_type`|`body`|`integer`|否|报警事件，监控点上墙码流类型，默认主码流，0 主码流；1 子码流|
|`camera_list`|`body`|`string`|是|报警联动监控点（多个中间以逗号隔开），从查询监控点列表v2接口获取返回参数indexCode|
|`wnd_list`|`body`|`string`|否|指定报警窗口uri，中间以逗号隔开（当有该参数时，将会直接以该参数中的窗口报警上墙，而不执行tvms内部的报警查找窗口的逻辑）,从获取电视墙大屏信息接口获取返回参数大屏的wnd_uri|
|`event_desc`|`body`|`string`|否|报警描述|

补充说明:
- 当小于5S时以服务端报警时间配置为准，默认35S

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.1.2 电视墙错误码|
|`msg`|`body`|`string`|否|返回信息|
|`data`|`body`|`object`|否|返回数据，成功时返回空|

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {}
}
```
