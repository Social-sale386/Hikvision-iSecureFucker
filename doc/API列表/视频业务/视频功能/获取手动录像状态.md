# 获取手动录像状态

> 接口说明
> 该接口用于根据手动录像任务ID，向存储接入服务下发查询手动录像状态请求, 设备存储暂不支持该能力。萤石设备不支持手动录像功能。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.5及以上版本|
|接口地址|`/api/video/v1/manualRecord/status`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/video/v1/manualRecord/status
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`cameraIndexCode`|`body`|`string`|是|监控点唯一标识，分页获取监控点资源接口获取返回参数cameraIndexCode|
|`taskID`|`body`|`string`|否|手动录像编号，从开始手动录像接口获取返回参数taskID，由于一个监控点同种存储类型同一时间只能进行一个手动录像任务，taskID不传入时，根据监控点编号和存储类型获取手动录像状态。|
|`type`|`body`|`integer`|否|下发的存储类型,0-中心存储 1-设备存储|

补充说明:
- 如果不传入，默认为0-中心存储

## 请求参数示例
```json
{
  "cameraIndexCode": "fee3d641f0e44ecCC9b9def2b423b7eb",
  "taskID": "1",
  "type": 0
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0-成功，其它-失败，参考附录E.1.1 视频应用错误码|
|`msg`|`body`|`string`|否|返回描述|
|`data`|`body`|`object`|否|返回数据|
|`+taskStatus`|`body`|`number`|否|手动录像状态：|

补充说明:
- 0：未接入 ；
- 1：正常执行 ；
- 2：异常

## 返回参数示例
```json
{
  "code": "0",
  "msg": "success",
  "data": {
    "taskStatus": 1
  }
}
```
