# 录像锁定与解锁

> 接口说明
> 通过向中心存储接入服务下发锁定/解锁指定编码器，指定时间段的录像。
> 进行录像锁定/解锁操作的用户需要配置相应监控点的录像回放权限，本接口会根据传入的userId进行权限过滤。
> 仅支持中心存储，暂不支持设备存储。

## 基本信息
|项目|值|
|---|---|
|接口版本|v1|
|适配版本|综合安防管理平台iSecure Center V1.4及以上版本|
|接口地址|`/api/video/v1/record/lock`|
|请求方法|`POST`|
|数据提交方式|application/json|

## 快速调用
```text
POST /api/video/v1/record/lock
```

## 请求参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`cameraIndexCode`|`body`|`string`|是|监控点唯一标识，分页获取监控点资源接口获取返回参数cameraIndexCode|
|`type`|`body`|`integer`|否|下发的存储类型。0-中心存储，1-设备存储。如果不传入，默认为0-中心存储|
|`startTime`|`body`|`string`|是|开始时间（IOS8601格式：yyyy-MM-dd’T’HH:mm:ss.SSSzzz +当前时区，例如北京时间：2018-07-26T15:00:00.000+08:00）|
|`endTime`|`body`|`string`|是|结束时间（IOS8601格式：yyyy-MM-dd’T’HH:mm:ss.SSSzzz +当前时区，例如北京时间：2018-07-26T15:00:00.000+08:00）|
|`lockTime`|`body`|`integer`|是|锁定时长，单位：秒，为0时表示解锁，取值范围：0-1576800000|

## 请求参数示例
```json
{
  "cameraIndexCode": "fee3d641f0e44ecCC9b9def2b423b7eb",
  "type": 0,
  "startTime": "2018-07-26T15:00:00.000+08:00",
  "endTime": "2018-07-26T15:00:00.000+08:00",
  "lockTime": 10
}
```

## 返回参数
|参数|位置|类型|必填|说明|
|---|---|---|---|---|
|`code`|`body`|`string`|否|返回码，0 – 成功，其他- 失败，参考附录E.1.1 视频应用错误码|
|`msg`|`body`|`string`|否||
|`data`|`body`|`object`|否|返回数据|
